16 要素框架

Prompt 优化：把一句话升级成 AI 需求规格说明书

一个优秀 Prompt 不只是告诉 AI「做什么」，而是说清楚角色、目标、上下文、流程、约束、输出格式和验收标准。下面这套 16 要素框架，适合写代码、系统设计、面试模拟、写作和 Agent Prompt。

标准 Prompt 框架

Prompt ├── Role / Goal / Context ├── Audience / Input / Task ├── Workflow / Thinking ├── Constraint / Output ├── Quality / Verification └── Iteration / Example / Persona / Success
可复制 可落地 可复用

为什么 Prompt 应该像需求规格说明书？

因为 AI 的输出质量，取决于输入信息是否完整、边界是否清晰、成功标准是否明确。

优秀 Prompt 的核心链路：你是谁（Role）→ 为什么做（Goal）→ 用什么方法做（Workflow）→ 输出什么（Output）→ 什么不能做（Constraint）→ 如何判断做好了（Evaluation）。

16 个核心模块

1. Role（角色）定义 AI 的职业、经验、领域、思维方式和价值观。
2. Goal（目标）明确最终要达成什么，而不是只描述一个动作。
3. Context（上下文）说明项目背景、技术栈、已有系统、限制和业务目标。
4. Audience（受众）告诉 AI 最终内容给谁看：面试官、CTO、用户或老师。
5. Input（输入）明确会提供哪些材料：源码、日志、简历、JD、SQL、图片或 PDF。
6. Task（任务）拆清楚要分析、设计、优化、生成、审核、调试还是规划。
7. Workflow（流程）规定执行步骤，避免 AI 自由发挥导致遗漏。
8. Thinking（思维模式）指定 MECE、第一性原理、STAR、PDCA、ReAct、5Why 等方法。
9. Constraint（约束）写清楚不能编造、不能泛泛而谈，必须符合生产环境等边界。
10. Output（输出格式）规定 Markdown、JSON、YAML、HTML、Mermaid 或固定章节结构。
11. Quality（质量要求）要求专业、完整、企业级、可维护、可扩展、符合工程规范。
12. Verification（校验）输出前自检是否遗漏、是否有 Bug、是否满足约束。
13. Iteration（迭代机制）信息不足时先提问；复杂任务分阶段确认。
14. Example（示例）提供 few-shot 示例，让输出风格和结构更稳定。
15. Persona（表达风格）控制语气：严谨、简洁、导师、CTO、面试官或学术论文。
16. Success Criteria（成功标准）定义什么叫完成：可运行、可上线、可放简历、可通过面试。

不同场景的侧重点

场景最重要的模块

标准模板

你是：{Role} 目标：{Goal} 背景：{Context} 受众：{Audience} 输入：{Input} 任务：{Task} 执行流程：{Workflow} 思维方式：{Thinking} 约束：{Constraint} 输出格式：{Output} 质量要求：{Quality} 结果校验：{Verification} 迭代机制：{Iteration} 示例：{Example} 表达风格：{Persona} 成功标准：{Success Criteria}

写 Prompt 的判断标准

是否说清楚最终目标，而不是只说一个动作？
是否提供足够上下文，让 AI 不需要猜？
是否规定输出结构，方便直接使用？
是否写明限制条件，避免编造和跑偏？
是否定义成功标准，让结果可验收？
