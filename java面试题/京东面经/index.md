## 8. Java 京东校招一面（ 30 分）

🟢 **简单** | 标签：Java, 校招, 京东, 后端, 面经

HashMap是基于哈希表实现的Map接口，允许null键值，非线程安全。其底层数据结构在JDK8后为**数组+链表+红黑树**。

核心原理是put过程：计算键的哈希值，经过扰动函数处理后，与数组长度取模确定存储桶位置。若桶为空，直接插入新节点；若发生哈希冲突，则以链表形式存储。当链表长度超过阈值（默认8）且数组长度大于64时，链表会转换为红黑树以提升查询效率。数组默认初始容量16，负载因子0.75，当元素数量超过容量*负载因子时，会触发**扩容**，将容量翻倍并重新哈希所有元素。

关键特性包括：允许null键值；查询、插入、删除的平均时间复杂度为O(1)；扩容时因重新哈希可能引发性能抖动；**线程不安全**，多线程下可能导致数据不一致。并发场景应使用`ConcurrentHashMap`，其通过分段锁（JDK7）或CAS+synchronized（JDK8）保证线程安全。与`Hashtable`相比，HashMap允许null且效率通常更高。

## 13. Java 京东（实习）一面

🟢 **简单** | 标签：Java, 京东, 实习, 后端, 面经, 校招

**Java集合框架**
ArrayList基于动态数组，适合随机访问；LinkedList基于双向链表，适合频繁插入删除。HashMap采用数组+链表+红黑树结构，通过hash值定位桶位置，解决冲突使用链地址法，JDK8后当链表长度>8时转为红黑树。ConcurrentHashMap采用分段锁或CAS操作保证线程安全。

**JVM内存与GC**
堆分为新生代（Eden、Survivor区）和老年代。GC主要针对堆，年轻代用复制算法，老年代用标记-清除或标记-整理。常见垃圾收集器如CMS（低停顿）、G1（分区收集）。类加载遵循双亲委派模型，自定义类加载器需继承ClassLoader。

**多线程与并发**
创建线程的方式有继承Thread、实现Runnable、Callable。线程池核心参数包括核心线程数、最大线程数、阻塞队列、拒绝策略。synchronized实现同步，volatile保证可见性。AQS和Lock接口提供了更灵活的锁机制。

**数据库与Redis**

## 15. Java京东商城一面（2年经验）

🟢 **简单** | 标签：Java, 社招, 京东, 后端, 面经, 京东商城

### JVM内存模型与垃圾回收机制
JVM内存主要分为堆、栈、方法区等。堆是对象实例分配区域，分为新生代（Eden、Survivor）和老年代。栈存储局部变量和方法调用。垃圾回收主要针对堆内存，算法包括标记-清除、复制、标记-整理等。新生代常用复制算法，老年代常用标记-整理。GC类型有Minor GC（新生代）、Major GC（老年代）和Full GC（整个堆），Full GC需尽量避免，因其会导致STW（Stop-The-World）停顿。

### Java并发编程核心
并发核心在于线程安全与锁机制。synchronized和ReentrantLock是常用锁，前者基于JVM实现，后者基于API，提供更多高级功能（如公平锁、条件变量）。volatile关键字保证变量可见性并禁止指令重排。并发集合如ConcurrentHashMap采用分段锁或CAS操作提高性能。线程池（ThreadPoolExecutor）通过复用线程减少开销，核心参数包括核心线程数、最大线程数、队列和拒绝策略。

### Spring框架核心原理
Spring核心是IoC（控制反转）和AOP（面向切面编程）。IoC通过容器管理对象依赖，实现解耦。AOP用于提取横切关注点（如日志、事务），通过动态代理实现。Bean生命周期包括实例化、属性注入、初始化（如@PostConstruct）和销毁。事务管理通过@Transactional注解声明，传播行为如REQUIRED、REQUIRES_NEW需根据业务场景选择。Spring Boot自动配置通过@EnableAutoConfiguration实现，简化开发。

### 数据库与性能优化
数据库优化重点在索引与查询。索引结构常用B+树，遵循最左前缀原则。避免索引失效，如对索引列使用函数或隐式类型转换。慢查询分析通过explain命令查看执行计划，关注type、key、rows等字段。事务隔离级别默认可重复读，通过MVCC（多版本并发控制）实现。分库分表在数据量大时考虑，水平拆分按规则将数据分布到多个库表。

### 项目经验与系统设计
项目中负责订单模块优化，发现高并发下数据库锁竞争严重。解决方案：1. 引入Redis缓存热点数据，降低数据库压力；2. 使用分布式锁（Redisson）控制并发下单；3. 异步化非核心流程（如通知），通过消息队列削峰。设计时遵循高可用原则，如降级、限流（Sentinel）。性能提升：QPS从500提升至3000，平均耗时降低60%。

## 26. Java 京东零售二面（技术一面) 高强度深底层拷打

🟡 **中等** | 标签：Java, 京东, 后端, 面经, 京东零售, 校招

**JVM 内存模型与 GC**
- 核心区域：堆（对象实例）、栈（线程私有，栈帧含局部变量表/操作数栈）、方法区（类信息、常量）。
- GC 算法：分代收集（年轻代用复制算法，老年代用标记-整理）。调优关键参数：-Xmx, -Xms, -Xmn, -XX:SurvivorRatio。理解 G1/ZGC 分区回收思想。

**并发编程**
- synchronized 原理：对象头 Mark Word 存储锁信息，经历偏向锁->轻量级锁->重量级锁升级。
- JUC 核心：AQS 抽象队列同步器是基础（ReentrantLock、Semaphore 内部实现）。线程池七大参数，重点理解 workQueue 和 RejectedExecutionHandler。volatile 保证可见性与禁止指令重排。

**集合框架**
- HashMap：数组+链表/红黑树。扩容机制（负载因子0.75），线程不安全。ConcurrentHashMap 在 JDK8 采用 synchronized + CAS 锁桶头节点。
- ArrayList 与 LinkedList 适用场景（随机访问 vs 频繁增删）。

**数据库与索引**
- InnoDB 索引结构：B+树，聚簇索引（主键）与二级索引（回表查询）。联合索引最左前缀原则。
- 事务隔离级别：InnoDB 默认可重复读（RR），通过 MVCC（多版本并发控制）实现，基于 undo log 版本链和 ReadView。
- 慢查询优化：explain 分析 type/key/rows，避免索引失效（如函数计算、类型转换）。

**总结**：我对 JVM 底层、高并发解决方案及数据库原理有深入理解，并能结合实际场景分析优化，具备扎实的后端开发基础。
