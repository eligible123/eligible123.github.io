# 网易Java面经

> 共 2 道面试题

## 1. Java 网易云音乐一面

🟢 **简单** | 标签：Java, 网易云音乐, 社招, 后端, 面经, 网易

网易云音乐作为高并发流媒体平台，其后端开发需重点考量以下技术要点：

首先在架构层面，系统采用微服务拆分（如推荐、曲库、社交等独立服务），通过Dubbo或gRPC进行服务间通信，并依赖Nacos或Zookeeper实现服务发现与配置管理。分布式事务方面，常采用最终一致性方案，如消息队列保证数据最终同步。

高并发场景下，缓存是关键。多级缓存策略（本地缓存+分布式Redis集群）用于应对热点歌曲及用户个性化推荐请求，通过布隆过滤器、热点数据探测防止缓存击穿。消息队列（如Kafka）用于削峰填谷，处理评论、动态等异步写入场景，同时保障消息顺序性。

数据库层面，MySQL集群按业务分库分表（如用户、歌单等垂直拆分，大表水平分片），结合读写分离提升吞吐。对于歌曲播放日志等海量数据，常采用时序数据库或日志聚合系统进行处理。

Java技术栈上，需深入理解JVM调优（如针对高并发场景的G1垃圾回收器配置）、并发编程（线程池定制、锁优化）及Spring Cloud生态组件。此外，对实时性要求高的场景（如歌词同步、直播弹幕），会结合WebSocket或Netty实现长连接通信。

最后，保障服务稳定性需依赖全链路监控（如Prometheus+Grafana）、限流降级（Sentinel）及容灾方案（如数据库主从切换、服务多活部署）。以上技术组合共同支撑起网易云音乐亿级用户的服务体验。

## 2. Java 网易服务端一面

🟢 **简单** | 标签：Java, 后端, 网易, 面经, 校招

网易Java服务端一面主要考察基础扎实度和系统性思维。首先，Java核心基础是重点，需熟练掌握集合框架（如HashMap底层原理、并发容器）、多线程（线程池参数、锁机制）、JVM内存模型与GC算法。其次，对数据库MySQL要理解索引优化、事务隔离级别及MVCC机制。中间件方面，Redis的缓存穿透/雪崩解决方案、数据结构应用场景是常见考点。框架层面，Spring的IoC与AOP原理、Bean生命周期需要清晰阐述。此外，分布式基础如CAP理论、分布式锁实现方案也应有所了解。面试时建议结合项目经验，用STAR法则描述技术选型与问题解决过程，展现工程化思维。保持回答结构化，先定义概念、再讲原理、最后结合实例，能更好地体现深度与广度。
