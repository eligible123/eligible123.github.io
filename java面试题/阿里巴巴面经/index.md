Java中的垃圾回收（GC）机制是JVM自动管理内存的核心，它通过识别和清除不再使用的对象来防止内存泄漏。其基本原理是基于可达性分析：从GC Roots（如栈帧中的局部变量、静态变量等）出发，遍历对象引用链，无法到达的对象即被视为垃圾。

主要的垃圾回收算法包括标记-清除、复制和标记-整理。标记-清除算法会产生内存碎片；复制算法适用于新生代，将存活对象复制到另一块空间；标记-整理则用于老年代，移动存活对象以消除碎片。

## 6. Java 阿里实习一面（30min）

🟢 **简单** | 标签：Java, 阿里巴巴, 实习, 后端, 面经

阿里Java实习一面通常时长30分钟，考察范围广泛但聚焦基础，核心是评估候选人的Java技术功底和学习潜力。以下是各模块的核心要点：

**1. Java基础与集合**
重点考察`HashMap`的底层原理（数组+链表/红黑树）、扩容机制、并发问题（线程不安全及`ConcurrentHashMap`的优化）。此外，`ArrayList`与`LinkedList`区别、`String/StringBuilder/StringBuffer`、`final`关键字、抽象类与接口等也是高频问题。

**2. JVM与内存管理**
核心包括JVM内存区域划分（堆、栈、方法区）、垃圾回收算法（如复制、标记-清除）、以及典型的垃圾回收器（如G1）。类加载机制（双亲委派模型）和`OOM`问题的排查思路也需掌握。

**3. 并发编程**
必须深入理解`synchronized`和`volatile`的底层实现与区别。`Lock`接口、线程池核心参数（如`corePoolSize`、`workQueue`）及其工作流程是重点。同时需掌握`ThreadLocal`原理、`JUC`工具类（如`CountDownLatch`、`CyclicBarrier`）以及死锁的产生条件与避免方法。

**4. 框架与中间件**
Spring框架中，IoC容器原理、AOP实现机制（动态代理）、Bean的生命周期与作用域是必考点。对于Spring Boot，需了解其自动配置原理。数据库方面，重点在于MySQL的索引结构（B+树）、SQL优化技巧、事务隔离级别及MVCC机制，以及Redis的常用数据类型与持久化策略。

**5. 其他**
通常会包含1-2道算法题（如链表、树相关），以及对项目经历的深挖，需清晰阐述技术选型、难点与解决方案。最后可能涉及分布式基础概念，如CAP理论。

**准备建议**：回答时应结合原理与示例，展现系统性知识结构。对于不熟悉的问题，坦诚说明并给出分析思路，体现逻辑思维与学习能力。

在蚂蚁金服的高并发场景下，`ConcurrentHashMap` 通过**分段锁（JDK7）** 或 **CAS + synchronized（JDK8+）** 实现高效线程安全。其核心设计是：将数据分为多个桶（Node数组），读操作通常无锁（volatile保证可见性），写操作仅锁住单个桶的链表头或红黑树根，从而在保证线程安全的同时极大提升了并发读写性能。

在实际业务中，合理设置初始容量和负载因子可减少扩容开销。此外，需注意其迭代器是弱一致性的，适用于允许并发修改但对实时一致性要求不高的缓存等场景。

## 20. Java 阿里巴巴（盒马）二面

🟡 **中等** | 标签：Java, 盒马, 校招, 后端, 面经, 阿里巴巴

JVM垃圾回收（GC）的核心目标是自动管理堆内存，回收不再使用的对象，防止内存泄漏。其理解主要围绕三点：回收哪些对象、如何回收、以及如何选择收集器。

**首先，判断对象“已死”是关键。** 主流JVM采用可达性分析算法。它从一组称为“GC Roots”的根对象（如活跃线程栈中的引用、静态变量等）出发，遍历对象引用链。无法被访问到的对象即被判定为可回收，避免了循环引用问题。

**其次，具体回收算法与收集器。** 标记-清除算法会产生内存碎片；复制算法适用于新生代（Eden区），高效但空间利用率不高；标记-整理算法则用于老年代，避免碎片。JVM将堆划分为新生代和老年代，并采用分代收集策略：新生代对象朝生夕死，使用复制算法；老年代对象存活率高，使用标记-清除或标记-整理算法。对应的收集器如ParNew（新生代）与CMS（老年代），或更先进的G1、ZGC，它们在吞吐量、停顿时间等指标上做了不同权衡。

**最后，在面试中，我会强调理解GC的实际意义。** 这并非纯粹的理论。例如，知道对象如何晋升老年代，就能理解为何要避免创建大量短生命周期的大对象；熟悉GC日志和工具（如jstat），是线上应用进行内存诊断和性能调优的基础。掌握这些，才能写出更健壮、高效、资源友好的Java后端代码。
