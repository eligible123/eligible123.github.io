# 滴滴出行Java面经

> 共 1 道面试题

## 1. Java 滴滴秋招面经

🟢 **简单** | 标签：Java, 校招, 后端, 滴滴, 面经

Java后端校招面试通常聚焦以下几个核心方向，以下是关键要点：

**一、Java基础与集合框架**
*   **HashMap底层原理**：数组+链表/红黑树结构，扩容机制（2倍）、哈希冲突解决、线程不安全特性。常对比Hashtable、ConcurrentHashMap。
*   **多线程与并发**：线程生命周期、synchronized与ReentrantLock区别、volatile关键字作用（可见性、禁止指令重排）、线程池核心参数（核心线程数、队列、拒绝策略）。
*   **JVM基础**：内存区域（堆、栈、方法区）、垃圾回收算法（标记-清除、复制、标记-整理）、常见收集器（如CMS、G1）、类加载机制（双亲委派）。

**二、计算机网络与操作系统**
*   **TCP/IP**：三次握手、四次挥手过程及状态、TCP与UDP区别、拥塞控制。
*   **HTTP/HTTPS**：HTTP1.1/2.0区别、HTTPS的SSL/TLS握手过程。
*   **操作系统**：进程与线程区别、进程间通信方式、死锁条件与预防。

**三、数据库与Redis**
*   **MySQL**：索引原理（B+树）、事务隔离级别、MVCC实现、慢查询优化。
*   **Redis**：数据类型与应用（如String缓存、ZSet排行榜）、持久化机制（RDB/AOF）、缓存穿透/雪崩解决方案。

**四、场景与框架**
*   **场景设计**：如设计一个短链系统、秒杀系统，考察系统设计、高并发与缓存策略。
*   **Spring框架**：IoC/AOP原理、Bean生命周期、常用注解。
*   **分布式基础**：CAP理论、分布式事务解决方案（如Seata）、消息队列作用（解耦、削峰）。

**五、算法与编码**
*   常考察：动态规划、二叉树遍历、链表操作、排序算法等。需熟练手撕代码，并注重边界条件与复杂度分析。

**建议**：回答时结合原理与项目实践，展现问题解决能力和学习深度。针对滴滴业务，可关注出行场景下的高并发、地理位置服务等相关技术点。
