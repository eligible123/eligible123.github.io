## 6. Java 小米研发（校招）二面

🟢 **简单** | 标签：Java, 求职, 后端, 面经, 小米, 校招

Java集合框架中，HashMap是核心数据结构，基于数组+链表+红黑树实现。其核心要点如下：

1. **底层结构**：采用数组+链表+红黑树（JDK1.8+）。数组每个位置是桶，哈希冲突时用链表存储，链表长度超过8且数组长度≥64时转为红黑树，提升查询效率。

2. **扩容机制**：默认初始容量16，负载因子0.75。当元素数量超过容量*负载因子时，触发扩容（容量翻倍），重新哈希分布元素。扩容时可能产生死循环（JDK1.7），1.8通过高低位链表优化。

3. **关键设计**：
   - **哈希扰动**：高16位与低16位异或，减少碰撞。
   - **树化阈值**：链表长度≥8且数组≥64时树化，避免频繁树化。
   - **并发替代**：线程不安全，需用ConcurrentHashMap（分段锁/CAS）。

4. **面试技巧**：
   - 对比HashSet（基于HashMap）、Hashtable（全表锁）。
   - 说明为什么String、Integer适合做Key（不可变、重写hashCode）。
   - 结合实际场景（如缓存、索引）说明使用注意事项。

**总结**：掌握HashMap的结构、扩容、并发问题及替代方案，能体现对集合原理和线程安全的理解。
