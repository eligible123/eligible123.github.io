# Agent面试题

## Agent/工具调用

### 1. 🟢 Agent 的核心组件有哪些？

LLM(规划+推理) + Memory(短/长期) + Tools(外部能力) + Planner + Executor + Reflection。关键区别是「循环执行」而非一次调用。

### 2. 🟢 ReAct 的执行流程是什么？

Thought → Action(选工具+参数) → Observation(工具结果) → Thought 循环，直到 Final Answer 或达步数上限。

### 3. 🟢 Function Calling 和 MCP 的区别？

FC 是模型↔工具的协议(OpenAI 起源)；MCP 是工具/数据源标准协议(Anthropic)，目标让工具跨模型/跨 Agent 复用。

### 4. 🟢 多 Agent 系统常见协作架构有哪些？

中心化(Orchestrator+Worker) / 去中心化(消息总线) / 分层(Manager→Specialist)。讲清「分配 / 避免循环 / 聚合」三件事。

### 5. 🟡 Agent 的规划能力怎么实现？

显式规划(先输出 plan 再执行) vs 隐式(边想边做)。常用 ReAct / Plan-and-Execute / ToT；复杂任务先 plan 后 reflection。

### 6. 🟢 Agent 的三种记忆机制分别是什么？

短期=对话上下文；工作=本任务中间状态；长期=向量库存历史/偏好。要会答各自适合的场景 + 数据结构。

### 7. 🔴 Agent 工具调用失败怎么处理？

重试(指数退避) + 工具替换/降级 + Reflection 改参数 + 兜底回复。强调「不要无限循环」，设 max_retry。

### 8. 🟡 怎么评估 Agent 的效果？

任务完成率 + 步数效率 + 工具调用正确率 + Token 成本 + 人工评分。端到端 + 单步双轨评估。

### 9. 🟢 LangChain 和 LangGraph 的区别？

LangChain 链式抽象，适简单工作流；LangGraph 显式 DAG/图，适复杂分支 + 循环 + 多 Agent。

### 10. 🔴 Agent 多轮执行出错怎么定位？

全链路 trace(每步输入/工具/输出) + LangSmith/LangFuse 可观测平台；按步看哪一步偏离预期。

### 11. 🟡 ReAct 比单纯 LLM 强在哪？

能调外部工具拿事实，多步推理时能校验中间步骤；适合需要现场查/算的任务。

### 12. 🟡 Agent 怎么决定调用哪个工具？

LLM 根据 prompt 中的 tool 描述 + 当前上下文 + few-shot 例子决策。好的 tool 描述 = 决策关键。

### 13. 🔴 Reflection 失败 3 次后怎么处理？

切换工具 / 降级到固定回复 / 转人工 / 给错误说明 + 重新提问引导。设 max retry 防死循环。

### 14. 🟢 Coze / 扣子是 Agent 平台还是工作流平台？

字节出品的 Agent 工作流平台，本质低代码可视化编排 + LLM 节点；和 Dify 同类，介于纯 Agent 和 BPM 之间。

### 15. 🟡 流式返回时如何插入非文本事件(工具调用/思考/错误)且不影响前端渲染？

用结构化事件流(event: text|tool_call|thinking|error)，前端按 event type 分别渲染。

### 16. 🔴 ReAct 在实际开发中如何避免「思考与行动脱节」？

tool 描述要精准 + Few-shot 示范 + 必要时先规划再 act + Reflection 校验上一步是否真做到。

### 17. 🟡 Agent 工具调用的 Schema 设计核心是什么？

名字精准(动宾) + 描述说清「何时调」+ 参数 schema 严格 + 加 example + 避免歧义和重叠。

### 18. 🟢 常见 Agent 框架及对比维度？

LangChain/LangGraph/AutoGPT/CrewAI/Coze/Dify。按「上手成本 / 扩展性 / 可观测性 / 适合场景」对比，至少能深讲 2 个。


## RAG/检索

### 19. 🟢 RAG 的整体流程是什么？

Query → 改写(可选) → Embed → 检索(向量+BM25 混合) → Rerank → 组装 context → LLM 生成。强调「检索为生成服务」。

### 20. 🟡 RAG 解决了大模型的什么核心问题？

幻觉、知识过时、私有数据无法访问。本质是把「事实」从模型权重里拆出来变成可更新的外部知识。

### 21. 🔴 文档切片粒度怎么定？切大切小各有什么风险？

太大→噪声多 + context 浪费；太小→语义不全。常用 500–1500 字，按语义边界(段/小节)切，加 overlap。

### 22. 🟡 向量召回与 BM25 各适合什么场景？为什么要混合？

向量懂语义/同义改写；BM25 对术语/精确匹配强。常做 hybrid + Rerank 取长补短。

### 23. 🟢 Embedding 模型怎么选？主流方案？

看场景(中英/通用/领域) + 维度(算力) + 最大输入 + benchmark(MTEB)。中文常用 BGE/M3E/E5。

### 24. 🟡 Rerank 在 RAG 里起什么作用？什么时候必须加？

召回阶段牺牲精度换召回，Rerank 用 cross-encoder 精排 Top N。文档质量参差或召回噪声多时必加。

### 25. 🟢 如何评估 RAG 的检索质量？常用指标？

Recall@K / Precision@K / MRR / NDCG；可加 LLM-as-judge 评忠实度。

### 26. 🔴 RAG 的幻觉怎么治理？

提高召回质量 + Prompt 约束「基于上下文回答」+ 校验生成能否在 context 找到依据 + cite 来源。

### 27. 🔴 长文档怎么处理？切片 / 摘要 / 父子文档怎么选？

短问题→直接切片；信息分散→摘要+索引；语义连贯重要→父子块。常组合用。

### 28. 🔴 多模态 RAG 怎么做？有什么坑？

文/图/表分别 embed → 统一/分库召回 → LLM 多模态理解。坑：模态对齐、混合排序、上下文长度。

### 29. 🟡 知识图谱和 RAG 怎么结合？适合什么场景？

实体识别后查图谱拿关系 + 向量检索拿描述。适合金融/医疗强关系场景，弥补纯文本 RAG 关系缺失。

### 30. 🔴 RAG 召回率低怎么排查？

query 改写质量 → 切片粒度 → embedding 适配度 → 索引参数 → 数据覆盖度，逐层定位。

### 31. 🔴 Top K 怎么定？太大太小各有什么问题？

太小漏召回；太大噪声 + token 浪费。常 K=5–10，配 Rerank 取 Top 3–5 送 LLM。

### 32. 🟡 RAG 上下文压缩有哪些方法？

LLMLingua 类 token 压缩 + 句子级抽取 + Rerank+截断 + Map-Reduce 归纳。

### 33. 🟡 与微调相比，RAG 主要解决了什么？

知识更新成本低 + 可解释(给来源) + 私有数据不入权重 + 动态可控。

### 34. 🟡 除了基础向量检索，还有哪些提升 RAG 检索质量的技术？

Query 改写/扩展、HyDE、混合检索、Rerank、父子块、上下文压缩、自适应 K、查询路由。

### 35. 🟢 如何全面评估一个 RAG 系统的性能？

检索 + 生成 + 端到端三层，再加时延 + token 成本 + 用户满意度。


## Prompt/思维链

### 36. 🟡 CoT 的原理和效果来源？

让模型显式输出中间步骤(「Let's think step by step」)激活推理能力。本质是用更多 token 换更好 reasoning。

### 37. 🟡 Few-shot vs Zero-shot 怎么选？

任务复杂/格式严格→Few-shot 给样例；任务清晰、模型能力强→Zero-shot 省 token。

### 38. 🔴 Prompt 注入怎么防范？

分隔符 + 角色绑定 + 输入过滤 + 输出审计 + 不让用户控指令边界 + 关键操作二次确认。

### 39. 🟢 一个好的 Prompt 模板包含哪些要素？

角色 + 任务 + 输入 + 输出格式 + 约束 + Few-shot 例子，结构化分段。

### 40. 🟡 怎么评估 Prompt 的优劣？

离线评估集 + 业务指标 + A/B 对比 + LLM-as-judge，要有客观标准。

### 41. 🔴 思维链失败怎么调整？

更详细的 Few-shot CoT 示例 + 拆任务 + 加 Reflection + 换更强模型 + 降低 Temperature。

### 42. 🟡 多轮对话的 Prompt 怎么设计？

system prompt 锚定角色 + 历史摘要压缩 + 关键信息槽位提取 + 防 prompt 漂移。

### 43. 🟢 Prompt Chaining 是什么？

把复杂任务拆成多个 prompt 串行(前一步输出当后一步输入)，比单个超长 prompt 更稳定可调试。

### 44. 🔴 结构化输出怎么保证？

JSON schema + Few-shot + Function Call/Tool Schema + 输出后 JSON 解析重试 + 严格模式(structured output)。

### 45. 🟡 上下文工程和提示词工程有什么区别？

Prompt 工程偏单次指令优化；上下文工程偏「喂给模型的所有 context」组织(系统 prompt+历史+RAG 召回+工具描述)。

### 46. 🔴 如何做自动化 Prompt 优化(A/B test / eval)？

评估集 + 候选 prompt → 跑分对比 → 自动迭代(用 LLM 改 prompt)。要防过拟合到评估集。


## 项目深挖/简历

### 47. 🟢 介绍最有挑战的 AI 项目，怎么讲？

选 1 个有 AI 含量(非套壳)+有指标+有踩坑的项目，按「业务背景→技术方案→关键难点→指标」4 段讲，控制 2–3 分钟。

### 48. 🟡 项目核心难点应该抓什么？

抓「非确定性」难点(评估难/数据难/对齐难/部署难)，不要泛泛说「工程量大」。

### 49. 🟡 选型为什么是这个方案？怎么答？

至少列 2 个候选 + 比较维度(成本/性能/可维护性/数据需求)，证明做过 trade-off。

### 50. 🟡 项目踩了哪些坑、怎么解决？

选 2–3 个具体坑(幻觉/召回低/推理慢/上下文超长)，每个讲「现象→根因→方案→效果」。

### 51. 🟢 关键指标 / 北极星指标怎么答？

北极星 = 真实业务指标(留存/转化/任务完成率)，配代理指标(召回率/CTR)。两者要能解释因果。

### 52. 🟡 上线后效果怎么衡量？

A/B 实验 + 离线指标 + 用户反馈三件套。不要只说「反馈好」，要给数。

### 53. 🟡 「如果重做你会怎么做」怎么答？

体现学习能力，选 1–2 个「现在视角看会做不同」的点(技术选型/数据策略)。

### 54. 🟢 「你在团队里的角色」怎么答？

用 STAR 说清「我具体做了什么、对成果有什么影响」，别说「参与」。

### 55. 🔴 构建数据集遇到哪些挑战？

数据清洗、标注一致性、长尾覆盖、版权/隐私、合成数据质量评估，挑 2–3 个深入。

### 56. 🟡 项目中如何进行 LLM 模型选择？

需求(能力/上下文/中英文)→候选(GPT/Claude/Qwen/DeepSeek)→评估(自有评估集)→trade-off(成本/合规/SLA)。

### 57. 🟢 简历上的 AI 项目，面试官到底想看什么？

有 AI 含量(非套壳) + 真问题 + 有指标 + 你做了关键决策。讲清「我」做了什么，不是「团队」。


## 工程/部署/Infra

### 58. 🟢 vLLM 的核心优化是什么？

PagedAttention(分页 KV cache) + Continuous batching(动态合批) + 量化 + prefix cache。吞吐比 HF 高数倍。

### 59. 🟢 模型量化和蒸馏的区别？

量化：降精度(FP16→INT8/INT4)，结构不变；蒸馏：教师→学生，结构变了。可叠加。

### 60. 🔴 端侧部署的挑战？

算力/显存/电耗受限。要小模型(SLM 1–7B) + 量化 + 推理引擎(MLC LLM/llama.cpp/ONNX Runtime)。

### 61. 🟢 分布式训练的常见策略？

DDP(数据并行) + Tensor 并行(切矩阵) + Pipeline 并行(切层) + ZeRO(优化器分片)。大模型常组合用。

### 62. 🟡 GPU 显存怎么优化？

量化 + KV cache 分页 + 梯度检查点 + 混合精度 + ZeRO + offload 到 CPU/NVMe。

### 63. 🔴 推理服务的限流和兜底？

令牌桶限流 + 优先级队列 + 排队 + 降级到小模型/缓存/规则回复。

### 64. 🟡 私有化部署的取舍？

数据安全 vs 维护成本 + 模型迭代速度。金融/政务/医疗强合规必须，中小企用 API 更经济。

### 65. 🟡 模型缓存怎么设计？

输入归一化 hash → 命中直接返回；语义近似可用 embedding 相似度。可降 30–70% 调用成本。

### 66. 🟢 常见推理框架有哪些？怎么选？

vLLM/TGI/TensorRT-LLM/SGLang/llama.cpp/MLC LLM。按部署场景(云端/边缘)和模型大小选。

### 67. 🟢 模型量化(INT8/INT4/FP8)的原理？

把权重/激活从 FP16/32 降到 INT/FP8，体积/算力小数倍。精度损失靠校准/QAT 控制。

### 68. 🔴 量化后的坑如何规避？

校准集要代表性 + 关键层(attention)保留高精度 + 对比量化前后效果 + 不在长上下文盲量化。


## 评估/幻觉/安全

### 69. 🟡 幻觉的根本原因？

模型本质是概率生成，没事实校验机制。预训练数据噪声 + 缺乏知识 + 外推超出能力。

### 70. 🔴 幻觉怎么治理(通用)？

RAG 拿事实 + Prompt 约束 + 自检 reflection + 关键输出人工/规则校验 + cite 来源。

### 71. 🟢 大模型评测有哪些标准？

MMLU/HumanEval/GSM8K/HellaSwag 等公共基准 + 业务任务自评 + LLM-as-judge + 红队测试。

### 72. 🟡 AI bias 怎么处理？

数据均衡 + 对齐阶段加 fairness 约束 + 上线后偏见监控 + 关键决策保留人工。

### 73. 🔴 越狱攻击怎么防？

Prompt 加防御指令 + 输入分类器 + 输出审核 + RLHF 对齐安全场景 + 持续红队迭代。

### 74. 🟡 可解释性 AI 在什么场景必须？

金融信贷 / 医疗诊断 / 司法 / 招聘等高风险高合规场景，决策需可追溯可解释。

### 75. 🟢 AIGC 水印是什么？

在生成内容中嵌入不可见可检测的标记，用于溯源 / 反伪造 / 版权。

### 76. 🔴 AI 偏见在招聘/信贷场景的风险？

历史数据偏见放大 → 歧视性决策 → 合规风险(GDPR/反歧视法) + 品牌风险。

### 77. 🟡 如何做输出内容审核 / 毒性检测？

关键词 + 分类器(toxic/violent/political) + LLM judge + 灰度策略 + 反馈迭代。


## Coding/手撕

### 78. 🟢 手撕 Transformer 简化版的关键点？

实现 Multi-Head Attention + LayerNorm + FFN 一个 block。关键：KV 投影、scale、mask、softmax。

### 79. 🟢 实现简易 self-attention 的公式与关键？

softmax(QKᵀ/√d)·V，三个线性层 + scaled dot-product。要能讲清 mask、numerical stability。

### 80. 🟢 实现一个 BPE 分词的步骤？

统计 byte-pair 频率 → 迭代合并最高频对 → 构建词表。重点 merge rule + encode/decode。

### 81. 🟢 手撕简易 RAG 流程？

load docs → chunk → embed → 存向量库 → query embed → top-K 检索 → 拼 prompt 调 LLM。

### 82. 🟢 手撕 ReAct Agent 框架？

loop：LLM 生成 thought+action → 解析 → 调 tool → 拿 observation → 喂回 LLM。终止条件 + max steps。

### 83. 🟡 滑动窗口算法怎么实现？

双指针维护窗口 [l,r]，扩 r 加入元素，触发条件后收 l。时间 O(n)。

### 84. 🔴 滑动窗口的结构体通常包含哪些字段？

l/r 指针 + 窗口统计(map/sum) + 当前最优解 + 边界判断标志。

### 85. 🟢 实现一个简单的工具调用？

定义 tool schema → LLM 输出 JSON 调用 → 解析参数调函数 → 返回结果给 LLM。

### 86. 🟢 高频 LeetCode 三大基础算法注意点？

二分注意边界，滑窗注意收缩条件，DP 注意状态定义+转移。准 Hot 100。

### 87. 🟢 链表/二叉树常见题型？

链表：反转/合并/环检测；树：遍历/层序/最近公共祖先/序列化。递归 + 双指针。
