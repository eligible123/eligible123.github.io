## 3. 2023A-工作安排

🟢 **简单** | 标签：华为OD, DP

# 工作安排问题 - 动态规划解法

## 问题分析

工作安排类问题通常要求在满足特定约束条件下，合理分配任务以获得最优结果（最大收益/最小代价）。核心思想是**将问题分解为子问题，通过状态转移逐步求解**。

## 解题步骤

**第一步：明确状态定义**
- 设 `dp[i]` 表示处理前 i 项工作时的最优解
- 或 `dp[i][j]` 表示在第 i 天/阶段，处于状态 j 时的最优解

**第二步：确定状态转移方程**
- 根据题意分析相邻状态间的递推关系
- 通常形式为：`dp[i] = max/min(dp[i-1] + cost[i], dp[i-k] + ...)`
- 考虑每项任务"选或不选"两种情况取最优

**第三步：设定边界条件**
- 初始化 `dp[0]` 或基础情况的值
- 处理特殊情况（如工作数为0、1时）

**第四步：确定遍历顺序**
- 正序或倒序遍历，确保计算 `dp[i]` 时其依赖状态已求解

## 代码模板

```python
def solve(jobs):
    n = len(jobs)
    dp = [0] * (n + 1)
    for i in range(1, n + 1):
        dp[i] = max(dp[i-1] + jobs[i-1], ...)  # 根据题意调整
    return dp[n]
```

## 复杂度分析

- **时间复杂度**：O(n) 或 O(n·k)，取决于状态转移
- **空间复杂度**：O(n)，可优化至 O(1)（滚动数组）

**关键点**：面试时先理清状态含义，再推导转移方程，最后考虑边界和优化。

2. 遍历字符串的每个字符：

### 题目分析  
本题要求模拟积木合并游戏：给定一列数字积木，当相邻数字相同时可合并为数字+1的新积木，并继续向左检查合并，直到无法合并为止。最终输出合并后的序列。  

### 解题思路  
1. **栈结构模拟**：使用栈存储积木，遍历输入序列。  
2. **合并逻辑**：  
   - 当前积木若与栈顶相同，弹出栈顶并将当前数字+1入栈（新积木可能继续与左侧合并）。  
   - 若不同，则直接入栈。  
3. **处理连锁合并**：新积木入栈后需循环检查是否与新的栈顶相同，直至无重复。  

### 代码实现（C++）  
```cpp
#include <iostream>
#include <vector>
#include <stack>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<int> nums(n);
    for (int i = 0; i < n; i++) cin >> nums[i];
    
    stack<int> stk;
    for (int num : nums) {
        while (!stk.empty() && stk.top() == num) {
            stk.pop();
            num++; // 合并后数字+1
        }
        stk.push(num);
    }
    
    // 反转栈得到顺序结果
    vector<int> res;
    while (!stk.empty()) {
        res.push_back(stk.top());
        stk.pop();
    }
    for (int i = res.size() - 1; i >= 0; i--) 
        cout << res[i] << " ";
    return 0;
}
```

### 时间复杂度  
每个积木最多入栈出栈一次，时间复杂度 **O(n)**，空间复杂度 **O(n)**。

**贪心策略如下**：
1.  **排序预处理**：将`cookingTime`升序排列，`patience`降序排列。此操作确保用时间最短的菜去匹配耐心最差的人，从而为后续匹配留出更多机会。

3.  **循环终止**：当任一指针越界时循环结束，计数即为最大分配数。

**复杂度**：排序主导时间复杂度，为O(n log n + m log m)，空间复杂度O(1)（假设原地排序）。该策略通过保证“最容易满足的菜给最可能失败的人”，系统性提升了匹配成功率，符合贪心算法的最优子结构特性。

1. **预处理**：计算前缀和数组，时间复杂度O(n)。

**核心要点**：
- **时间优化**：将区间和查询从O(n)降至O(1)。
- **空间权衡**：需额外O(n)空间存储前缀和数组。
- **适用场景**：频繁查询区间和或与子数组和相关的优化问题。

**核心思路：** 利用**栈（Stack）**数据结构来模拟过程。

**解题步骤：**

1. **初始化一个栈**，用于存储玛丽走过的路径字符。

2. **遍历字符串中的每个字符**：
   - 若为普通字符，直接压入栈中（表示玛丽向前走一步）
   - 若遇到特殊规则字符（如 `#`），则从栈中弹出一个元素（表示回退一步）
   - 若遇到重复规则字符（如连续相同字符触发特殊效果），按题目要求处理

3. **遍历结束后**，栈中剩余的字符即为玛丽最终停留的位置状态。

**关键点：**
- 栈的**后进先出（LIFO）**特性完美匹配"前进一步再回退一步"的场景
- 需注意边界条件：回退时栈为空的处理
- 时间复杂度 **O(n)**，空间复杂度 **O(n)**

**示例代码框架：**
```python
def solve(s):
    stack = []
    for c in s:
        if c == '#':  # 回退
            if stack:
                stack.pop()
        else:          # 前进
            stack.append(c)
    return ''.join(stack)
```

**总结：** 本题本质是**栈的经典应用**，与"括号匹配"、"路径简化"等问题同属一类，关键在于识别出"前进与回退"对应栈的"入栈与出栈"操作。

## 34. 2024D-计算三叉搜索树的高度

🟢 **简单** | 标签：华为OD, DFS

计算三叉搜索树的高度可以采用深度优先搜索（DFS）递归求解。三叉搜索树每个节点包含三个子节点：左子节点（值小于当前节点）、中子节点（值等于当前节点，用于处理重复值）和右子节点（值大于当前节点）。高度定义为从根节点到最远叶子节点的最长路径上的节点数。

核心算法步骤如下：
1. 定义递归函数`getHeight(node)`，若节点为空，返回0（表示高度为0）。
2. 递归计算左子树高度`left_height`、中子树高度`mid_height`和右子树高度`right_height`。
3. 取三者中的最大值，并加1（当前节点自身高度），即返回`max(left_height, mid_height, right_height) + 1`。

时间复杂度为O(n)，其中n为节点数，因为每个节点仅被访问一次。空间复杂度为O(h)，h为树高，主要由递归调用栈深度决定，最坏情况下树退化为链状时h=n。

此方法简洁高效，适用于面试场景，体现了对树结构和递归算法的掌握

## 39. 2023A-删除重复数字后的最大数字

🟡 **中等** | 标签：华为OD, 单调栈

# 删除重复数字后的最大数字

## 解题思路

这道题的核心是**在保证每个数字只出现一次的前提下，让高位数字尽可能大**，因此采用**单调栈**思想。

**关键策略：** 遍历数字时，若当前数字比栈顶大，且栈顶数字在后面还会出现，则弹出栈顶，放入更大的数字。

## 算法步骤

1. **预处理：** 统计每个数字的剩余出现次数（`count`数组）
2. **单调栈遍历：**
   - 若当前数字已在栈中，直接跳过
   - 当栈非空且满足：栈顶 < 当前数字 + 栈顶数字后续还会出现，则弹出栈顶
   - 将当前数字入栈，标记已使用，剩余次数减1
3. **拼接结果：** 栈中元素即为答案

## 复杂度分析

- **时间复杂度：** O(n)，每个元素最多入栈出栈一次
- **空间复杂度：** O(1)，栈和标记数组大小固定（最多10个数字）

## 核心代码逻辑

```python
def maxNumber(s):
    count = {}  # 统计剩余出现次数
    for c in s:
        count[c] = count.get(c, 0) + 1
    
    stack = []
    used = set()
    
    for c in s:
        count[c] -= 1
        if c in used:
            continue
        while stack and stack[-1] < c and count[stack[-1]] > 0:
            used.remove(stack.pop())
        stack.append(c)
        used.add(c)
    
    return ''.join(stack)
```

## 总结

本题本质是**带约束的单调栈问题**，难点在于判断何时可以弹出栈顶元素——需要同时满足"栈顶更小"和"后续还能补回"两个条件。

# 猴子爬山

## 问题描述
一只猴子要爬 n 级台阶的山，每次可以跳 1 级或 3 级，问有多少种不同的爬山方式？

## 解题思路

**算法选择**：动态规划（DP）

**状态定义**：`dp[i]` 表示爬到第 i 级台阶的方法数

**状态转移方程**：`dp[i] = dp[i-1] + dp[i-3]`
- 到达第 i 级，可以从第 i-1 级跳 1 步，或从第 i-3 级跳 3 步
- 总方法数 = 两种来源之和

**初始条件**：
- `dp[0] = 1`（地面，1种方式：不动）
- `dp[1] = 1`（只有 1 种：跳1级）
- `dp[2] = 1`（只有 1 种：跳1级两次）
- `dp[3] = 2`（跳3级 或 跳1级三次）

**边界处理**：注意 `i >= 3` 时才累加 `dp[i-3]`

## 代码实现

```cpp

using namespace std;

int main() {
    int n;
    cin >> n;
    
    // 处理边界
    if (n <= 2) {
        cout << 1 << endl;
        return 0;
    }
    
    int dp[n + 1];
    dp[0] = 1;
    dp[1] = 1;
    dp[2] = 1;
    
    for (int i = 3; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 3];
    }
    
    cout << dp[n] << endl;
    return 0;
```

## 65. 2023B-寻找符合要求的最长子串

🟡 **中等** | 标签：华为OD, 不定滑窗

# 2023B-寻找符合要求的最长子串

## 解题思路

本题采用**不定滑动窗口**（双指针）算法，核心思想是维护一个动态变化的窗口，在满足条件时更新最大长度。

### 算法步骤

1. **初始化**：定义左指针 `left = 0`，最大长度 `maxLen = 0`，使用哈希表记录窗口内字符状态

2. **扩展窗口**：右指针 `right` 从左到右遍历字符串，将当前字符加入窗口并更新状态

3. **收缩窗口**：当窗口状态不满足题目要求时，循环移动 `left` 指针，移出字符直到满足条件

4. **更新答案**：窗口满足条件时，更新 `maxLen = max(maxLen, right - left + 1)`

### 代码模板

```python
def longest_substring(s):
    left = 0
    max_len = 0
    window = {}  # 记录窗口状态
    
    for right in range(len(s)):
        # 扩展窗口，更新状态
        window[s[right]] = window.get(s[right], 0) + 1
        
        # 不满足条件时收缩窗口
        while not_valid(window):
            window[s[left]] -= 1
            if window[s[left]] == 0:
                del window[s[left]]
            left += 1
        
        # 满足条件时更新答案
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

### 复杂度分析

- **时间复杂度**：O(n)，每个元素最多被访问两次
- **空间复杂度**：O(k)，k为字符集大小

### 关键点

- 滑动窗口本质是**双指针的优化**
- 不定滑窗的窗口大小根据条件动态变化
- 注意边界条件和窗口状态的维护

**核心思路**：这是一个典型的最大化最小值问题，可以通过二分查找解决。将问题转化为：判断给定一个距离d，能否选出至少M个点使得任意相邻两点距离≥d。由于d越大越难满足，具有单调性，因此可以对d进行二分搜索，找到最大的可行d。

**算法步骤**：
1. 对点坐标排序。
2. 二分查找的左边界为0（最小可能距离），右边界为L（或最大坐标差）。
3. 在二分过程中，对每个候选距离d，使用贪心策略检查：从第一个点开始，依次选取下一个与当前点距离≥d的点，统计能选的点数。

5. 重复直到左边界大于右边界，最后得到的边界即为最大最小距离。

**复杂度**：排序O(N log N)，二分查找O(log L)，每次检查O(N)，总体复杂度O(N log N + N log L)。

🟡 **中等** | 标签：华为OD, BFS

# 最长广播响应

## 题目理解

在网络拓扑中，一个节点发送广播消息，消息会沿网络逐跳传播。需要找出**最后一个收到消息的节点**及其**响应时间**（即图中最长的最短路径）。

## 核心思路

本质是求**无权无向图中单源最短路径的最大值**，BFS 天然适合解决此类问题。

## 解题步骤

1. **建图**：使用邻接表存储网络拓扑
2. **BFS遍历**：从源节点开始，逐层扩展，记录每个节点的层级（即距离）
3. **取最大值**：遍历结束后，最大距离即为最长响应时间

## 关键代码逻辑

```python
from collections import deque, defaultdict

def longest_broadcast(n, edges, start):
    # 1. 建图
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    
    # 2. BFS
    visited = {start}
    queue = deque([(start, 0)])
    max_dist = 0
    
    while queue:
        node, dist = queue.popleft()
        max_dist = max(max_dist, dist)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, dist + 1))
    
    return max_dist
```

- **时间复杂度**：O(V + E)，每个节点和边仅访问一次
- **空间复杂度**：O(V + E)，存储图和访问标记

## 面试要点

1. 明确这是**无权图最短路径**问题，用 BFS 而非 Dijkstra
2. 注意处理**图不连通**的情况
3. 区分"最长响应"是取所有最短路径的**最大值**，而非最长路径（那是 NP 问题）

## 87. 2023B-第N个排列

🟡 **中等** | 标签：华为OD, 回溯

**问题概述：**
给定数字 n 和 k，求由数字 1 到 n 所有可能排列组成的序列中，按字典序排列的第 k 个排列。

**核心思路：**
不采用回溯法暴力枚举所有排列（时间复杂度 O(n!)，效率极低）。而是利用排列的数学规律直接计算。关键在于，当确定了第一个数字后，后续的排列数固定为 (n-1)!，因此可以通过“除法定位”逐步确定每一位。

**算法步骤：**
1.  **预处理阶乘：** 计算从 0! 到 (n-1)! 的阶乘数组。
2.  **循环定位：** 从最高位（第0位）开始确定数字。维护一个可用的数字列表（初始为 `[1, 2, ..., n]`）。
    *   计算当前位的索引：`index = (k-1) // (n-1)!`。该索引对应当前可用数字列表中的数字。
    *   将该数字从列表中移除，加入结果字符串。
    *   更新 k：`k = k - index * (n-1)!`。
    *   n 减 1，继续处理下一位，直到所有位确定。
3.  **边界处理：** 当 k=1 时，剩余数字按升序直接拼接即为答案。

**复杂度分析：**
*   **时间复杂度：** O(n^2)，主要消耗在循环中

## 99. 2024D-CPU算力分配

🟡 **中等** | 标签：华为OD, 哈希集合

**问题描述**
给定一组CPU核心的算力值和一组待执行任务的算力需求，将每个任务分配到一个核心上，使得任务在核心上执行。任务必须满足其算力需求，即分配到的核心算力值需大于等于任务需求。目标是最大化成功分配的任务数量。

**核心思路与算法**
1.  **问题建模**：此问题是经典的“资源分配”或“匹配”问题。可抽象为：在满足每个任务的最低算力要求的前提下，尽可能多地分配任务。
2.  **算法选择**：采用**贪心算法**可以高效求解。
    *   **排序**：将CPU核心的算力值和任务的算力需求分别进行**降序排序**。这样能优先将高算力任务匹配给高算力核心，提高整体匹配成功率。
    *   **双指针遍历**：使用两个指针分别遍历排序后的核心列表和任务列表。
        *   当当前核心的算力 `>=` 当前任务的需求时，匹配成功，两个指针均向后移动，匹配计数加1。
        *   若不满足，则只移动核心指针，尝试为该任务寻找算力更高的下一个核心。
    *   重复此过程，直到遍历完其中一个列表。

**哈希集合的潜在应用**
若题目存在额外约束（如每个任务有标签，要求同一标签的任务不能分配到特定的核心组），则可以在贪心过程中引入**哈希集合**来快速查询和过滤不符合条件的分配，确保解的有效性。

**总结**
该解法的关键在于通过**降序排序**结合**贪心策略**，确保在每一次分配中都做出局部最优选择（用当前能满足任务的最强核心去匹配），从而逼近全局最优解。该算法时间复杂度主要由排序决定，为 `O(N log N + M log M)`，其中N和M为核心与任务的数量，效率很高，适合面试场景下的解答。

## 112. 2024D-回转寿司

**核心解法：单调栈**。我们维护一个单调递减栈，栈中存放寿司的索引。为了模拟环形，将数组逻辑上拼接为原数组的两倍长度（通过取模索引实现）。

**具体步骤**：
1.  初始化结果数组 `ans` 和栈 `stack`。
2.  **从后往前**遍历 `2*n` 次（`i` 从 `2*n-1` 到 `0`）。使用 `idx = i % n` 获取原始索引。
3.  当栈不为空，且当前寿司价格 `prices[idx]` 大于等于栈顶索引对应的寿司价格时，循环弹出栈顶。这保证了栈内元素对应的价格单调递减。
4.  如果栈为空，说明当前寿司到传送带一圈结束都没遇到更贵的，`ans[idx]` 记为 `-1`。否则，栈顶索引指向的就是下一个更贵寿司，`ans[idx]` 记为 `(栈顶索引 - idx + n) % n`。
5.  将当前索引 `idx` 压入栈中。

## 113. 2024D-园区参观路径

🟡 **中等** | 标签：华为OD, DP

本题是典型的路径优化问题，可采用动态规划（DP）解决，类似于旅行商问题（TSP）的变种。核心思路是通过状态压缩DP，高效计算遍历所有园区景点的最短路径。

首先，问题可抽象为：给定N个景点和它们之间的距离，求访问所有景点恰好一次并返回起点的最短路径。定义状态dp[mask][i]，其中mask是一个二进制数，表示已访问景点的集合，i表示当前所在景点。状态值dp[mask][i]表示从起点出发，访问了mask中所有景点，最终停在景点i的最短距离。

状态转移方程为：对于每个状态dp[mask][i]，遍历所有未访问的景点j，更新dp[mask | (1<<j)][j] = min(dp[mask | (1<<j)][j], dp[mask][i] + dist[i][j])。其中dist[i][j]是景点i到j的距离。

初始条件：dp[1<<start][start] = 0，表示从起点开始，距离为0。最终答案需遍历所有i，取dp[(1<<N)-1][i] + dist[i][start]的最小值，确保返回起点。

此算法时间复杂度为O(N^2 * 2^N)，适用于景点数N较小（如N≤15）的场景。面试中可强调空间优化技巧，如滚动数组减少内存占用，并讨论实际应用中如何预处理距离矩阵以提升效率。总结：DP通过状态压缩将问题分解为子问题，逐步构建最优路径，是解决小规模路径规划的高效方法。

```python
    liberties = set()

                if board[nx][ny] == 0:

    return len(liberties)
```

推荐使用**BFS（广度优先搜索）**求解，因BFS天然适合无权图的最短路径问题。若需遍历所有可能状态（如收集多类宝藏的路径组合），则结合**状态压缩BFS**或**DFS回溯**。

**解法步骤**  
1. **建模**：将地图抽象为二维网格，每个格子视为图节点，相邻可通行格子连边。  
2. **状态定义**：若需收集宝藏，状态可为 `(行, 列, 已收集宝藏集合)`，用二进制表示宝藏集合。  
3. **BFS遍历**：  
   - 起点入队，标记初始状态。  
   - 每次取出队首状态，尝试向上下左右移动。  
   - 若新位置无障碍且状态未访问，则更新并入队。  
   - 遇到宝藏时更新收集集合。  
4. **终止条件**：找到出口且满足宝藏收集条件时，返回当前步数。  

**复杂度优化**  
- 时间复杂度：O(行×列×2^宝藏数)，适用于宝藏数≤10的场景。  
- 空间优化：使用位运算压缩状态，避免重复访问。  

**扩展场景**  
- 若允许多次经过同一格子，需用三维数组记录状态。  
- 若需输出具体路径，可在BFS中记录父节点并反向回溯。

这道题通常要求将给定整数n分解为若干2的幂次和3的倍数之和，例如将n表示为 a*2 + b*3 的形式，且a、b均为非负整数。核心是求解a和b的值。

解题思路是模拟枚举。从尽可能多的使用3开始，逐步减少3的个数，检查剩余部分是否能被2整除。因为2和3的组合覆盖了所有大于1的整数（除1本身外），我们只需找到一组可行解即可。需要处理特殊输入n=1的情况，此时无法分解，应按约定输出-1 -1。

算法步骤：
1. 首先处理 n=1 的特殊情况。
2. 初始化一个变量k，表示尝试使用k个3。
3. 从 k=n/3（最大可能）开始向下枚举到0。
4. 对于每个k，计算剩余值 r = n - 3*k。
5. 如果 r >=0 且 r % 2 == 0，则找到了一组解：a = r/2， b = k。
6. 输出 a 和 b。如果枚举完所有k都没有找到，则输出-1 -1。

该方法时间复杂度为O(n)，空间复杂度为O(1)，通过遍历保证找到解，逻辑清晰且易于实现。在面试中，重点在于阐明枚举策略和边界条件的处理。

## 135. 2024D-整数对最小和

🟡 **中等** | 标签：华为OD, 模拟

针对“整数对最小和”问题，核心思路是通过排序后使用双指针贪心求解。

**解题步骤：**
1.  **排序**：对给定的两个整数数组分别进行升序排序。
2.  **双指针遍历**：使用两个指针 `i` 和 `j` 分别指向两个数组的起始位置。
3.  **贪心选择**：在满足 `arr1[i] + arr2[j] < k` 的前提下，计算当前和并更新全局最小值。由于数组已排序，若当前和小于 `k`，则固定 `i` 不变，增加 `j` 可能得到更小的和（因为和会增大），因此应移动指向较小元素的指针（通常是比较 `arr1[i]` 和 `arr2[j]`，移动值较小的那一方）以尝试获得更优解。
4.  **终止条件**：当任一指针超出数组范围时结束。

**时间复杂度**：主要消耗在排序上，为 `O(n log n + m log m)`，遍历部分为 `O(n + m)`。

**适用场景**：该方法能高效找到满足条件的最小和，避免了暴力枚举所有组合，在面试中展示了排序与贪心算法的结合应用。

## 139. 2024D-最大N个数与最小N个数的和

🟡 **中等** | 标签：华为OD, 哈希集合

# 最大N个数与最小N个数的和

## 题意理解

给定一个整数数组和数值N，需要求出数组中**最大的N个数**与**最小的N个数**的和。需要注意的是，如果数组中存在重复元素且要求去重，则最大和最小N个数可能有交集，此时需要使用哈希集合处理。

## 解题思路

### 方法一：排序法（推荐）

**核心步骤：**
1. 将数组进行升序排序
2. 取前N个元素（最小N个数）和后N个元素（最大N个数）
3. 求和返回

**时间复杂度：** O(nlogn)  
**空间复杂度：** O(1)

### 方法二：堆/优先队列

- 使用最小堆取最大的N个数
- 使用最大堆取最小的N个数
- 适合数据流场景或只需要部分排序的情况

### 方法三：哈希集合处理去重

当需要去重时：
1. 将数组元素加入哈希集合去重
2. 转为排序数组
3. 取最小N个和最大N个求和

## 代码示例

```cpp
int solution(vector<int>& nums, int n) {
    sort(nums.begin(), nums.end());
    int sum = 0;
    for (int i = 0; i < n; i++) {
        sum += nums[i];              // 最小N个
        sum += nums[nums.size() - 1 - i]; // 最大N个
    }
    return sum;
}
```

## 边界情况

- 数组长度不足2N时，取所有元素
- 去重后元素不足2N时，取实际存在的元素

实现时可用单次循环，时间复杂度为O(n)，空间复杂度为O(1)。这种贪心思路清晰且高效，适合面试场景中展示对算法设计的理解。

**2. 内存分配（Allocate）**

**3. 内存释放（Free）**

**4. 扩展与回收**

**优势**：分配/释放效率高，无外部碎片。**局限**：存在内部碎片（块大小固定），且需要管理多档位大小以适应不同对象。该设计适用于嵌入式系统、高性能服务器等对内存管理效率要求严格的场景。

回溯算法的核心是深度优先搜索与状态回溯。具体步骤如下：

# 观看文艺汇演 - 解题思路

## 问题分析

这是一道典型的**区间调度/活动选择问题**，核心是在有限时间内选择观看节目，使观看的节目数量最多或获得的总收益最大。

## 解题思路

**方法一：贪心算法（节目数最大化）**
- 将所有节目按**结束时间**从小到大排序
- 从第一个节目开始，依次选择与已选节目不冲突的下一个节目
- 贪心策略：每次都选结束最早且不冲突的节目，为后续留出更多时间
- 时间复杂度：O(nlogn)

**方法二：动态规划（总时长/收益最大化）**
- **状态定义**：`dp[i]` 表示前 i 个时间段内最多能观看的节目数
- **状态转移**：对于当前节目，选择看或不看
  - 不看：`dp[i] = dp[i-1]`
  - 看：`dp[i] = dp[prev] + 1`，其中 `prev` 是与当前节目不冲突的最晚节目
- 时间复杂度：O(nlogn)（二分查找优化）

## 核心要点

1. **排序**是关键预处理步骤
2. 贪心适用于"最大化数量"，DP适用于"最大化权值/满足约束"
3. 注意边界条件：节目间隔是否需要休息时间、能否同时观看等

## 总结

本题考查对经典贪心/DP模型的掌握，解题关键在于**合理排序**和**正确建模状态转移**。

## 180. 2023B-陷阱方格

🔴 **困难** | 标签：华为OD, DFS&BFS

**问题描述**
在一个 M×N 的网格中，存在起点（S）和终点（E），以及部分“陷阱”方格。当走到陷阱方格后，后续所有移动都会自动“传送”到与上一个安全位置对称的相反方向方格（例如，从 (x,y) 安全移动到 (x+1,y) 踩中陷阱，则下一步移动方向若为 (dx,dy)，实际会到达 (x+1-dx, y-dy)）。目标是找到从 S 到 E 的最少步数。

**核心思路**
1.  **状态定义**：使用三维状态 `(x, y, trapped)`，其中 `(x,y)` 为坐标，`trapped` 为布尔值，表示当前是否处于“陷阱生效”状态。
2.  **搜索算法**：采用 **BFS** 进行逐层搜索，以确保找到最短路径。
3.  **状态转移**：
    *   **正常移动**：如果当前不在陷阱状态，向四个方向尝试移动到相邻非墙壁格子 `(nx, ny)`。如果 `(nx, ny)` 是陷阱，则下一状态为 `(nx, ny, true)`；否则为 `(nx, ny, false)`。
    *   **陷阱生效移动**：如果当前在陷阱状态，则对于每个方向 `(dx, dy)`，实际移动目标为 `(x - dx, y - dy)`。如果该目标在网格内且不是墙壁，则移动到 `(x - dx, y - dy)`，陷阱状态重置为 `false`（因为已执行一次“对称”移动）。
4.  **边界与记录**：使用集合记录已访问的 `(x, y, trapped)` 状态，避免重复搜索。

**关键点**
*   **状态机**：将陷阱的“激活”和“生效”过程建模为一个二元状态机，是解题的核心。
*   **BFS 适用性**：问题要求最少步数，且每一步的代价为1，BFS 能自然保证第一次到达终点即为最短路径。
*   **复杂度**：时间复杂度为 O(M×N×2)，空间复杂度为 O(M×N×2)，在合理网格范围内可行。

## 186. 2024D-二叉树的广度优先遍历

二叉树的广度优先遍历（BFS）的核心思想是**逐层访问节点**，通常使用**队列**实现。

其基本流程如下：
1.  **初始化**：创建一个队列，并将根节点入队。
2.  **循环处理**：当队列不为空时，执行循环：
    *   从队列头部取出一个节点，访问它（例如打印）。
    *   如果该节点的左子节点存在，则将其加入队尾。
    *   如果该节点的右子节点存在，则将其加入队尾。
3.  **终止**：队列为空时，遍历结束。

*   **数据结构**：队列（FIFO）是保证“先被发现的先被访问”的关键。
*   **时间复杂度**：O(n)，其中n为节点数，每个节点入队和出队一次。
*   **空间复杂度**：O(w)，w为二叉树的最大宽度，即队列的最大长度。
*   **应用场景**：常用于求树的最小深度、按层序列化/反序列化、寻找无权图最短路径等需要逐层处理的问题。

3. 遍历数组索引 `i`（从 0 到 `n-2`），更新 `prefix += nums[i]`，此时左部分和为 `prefix`，右部分和为 `total - prefix`。
4. 计算当前差值 `abs(prefix - (total - prefix))`，更新 `maxDiff`。
5. 返回 `maxDiff`。

**复杂度：** 时间复杂度 O(n)，空间复杂度 O(1)。通过单次遍历和前缀和累加，避免了重复计算，高效求解最大差值。

## 188. 2024D-分配土地最大面积

🔴 **困难** | 标签：华为OD, 贪心

本题本质是“最大化最小值”问题，可采用“贪心 + 二分”算法解决。

核心思路是：猜测一个可能的单块面积S，并验证能否将土地划分出至少K块面积不小于S的小块。若能，则S可尝试增大；若不能，则S需减小。通过二分法快速锁定最大的可行S。

具体步骤如下：
1.  **确定二分范围**：下界为0（或1），上界为初始土地面积。在此范围内进行二分查找。
2.  **贪心验证**：对于当前猜测的面积S，采用贪心策略计算能划分出多少块。从左到右、从上到下扫描土地，尽可能地划分出长宽均能满足面积要求的小矩形。统计划分出的块数。
3.  **调整边界**：若划分出的块数 ≥ K，说明S偏小或可行，记录当前S并尝试更大的值（调整下界）；否则S过大，调整上界。
4.  **循环直至收敛**：当二分范围缩至足够小时，即可得到最大面积。

该解法时间复杂度为 O(log(A) * (L*W / A))，其中A为面积范围，L、W为土地长宽。贪心验证步骤保证了在保证块数的前提下，尽可能为每块预留足够空间，从而最大化可行的最小面积。

## 190. 2024D-变换最小字符串

🔴 **困难** | 标签：华为OD, 贪心

本题要求通过有限次相邻字符交换，生成字典序最小的字符串。核心思路是采用贪心算法，每次在交换机会允许的范围内，选择最小字符提前到当前位置。

**算法步骤如下：**
1. 从左到右遍历字符串，设当前位置为i，剩余交换次数为k。
2. 在区间[i, min(n-1, i+k)]内寻找字典序最小的字符，记录其位置j。
3. 将该字符通过k'次相邻交换（k' = j - i）移至位置i，更新剩余交换次数k -= k'。
4. 重复上述过程直至遍历完成或交换次数耗尽。

**复杂度分析：** 时间复杂度O(n*k)，其中n为字符串长度，k为最大交换次数。

**总结：** 该算法的贪心策略在于每次决策都使当前前缀尽可能小，从而保证最终结果最优。需注意边界条件，如交换次数不足时提前终止。

## 199. 2024D-最长的指定瑕疵度的元音子串

🔴 **困难** | 标签：华为OD, 不定滑窗

## 解题思路

**题意理解**：在给定字符串中，找到一个最长子串，使其恰好包含指定数量（瑕疵度）的非元音字符。

**核心算法：不定长滑动窗口**

```
def solve(s, k):
    vowels = set('aeiouAEIOU')
    left = 0
    count = 0  # 窗口内非元音字符数
    max_len = 0
    
    for right in range(len(s)):
        # 右边界扩展
        if s[right] not in vowels:
            count += 1
        
        # 收缩左边界（非元音超标时）
        while count > k:
            if s[left] not in vowels:
                count -= 1
            left += 1
        
        # 满足条件时更新答案
        if count == k:
            max_len = max(max_len, right - left + 1)
    
    return max_len
```

**关键点**：

| 要素 | 说明 |
|------|------|
| 窗口条件 | `count == k` 时更新答案 |
| 收缩时机 | `count > k` 时移动左指针 |
| 时间复杂度 | O(n)，每个元素最多进出窗口一次 |
| 空间复杂度 | O(1) |

**面试要点**：解释清楚为什么用不定滑窗——因为目标是找"最长"子串，窗口可伸缩；瑕疵度作为约束条件控制窗口大小。

该方法时间复杂度为O(n)，空间复杂度为O(n)，适合处理大规模时间序列数据。面试中，可强调前缀和优化区间

## 203. 2024D-灰度图恢复

🔴 **困难** | 标签：华为OD, 模拟

**问题分析**  
本题要求恢复一张可能存在噪声或损坏的灰度图像。核心是利用已知像素信息，通过合适的算法推断缺失或异常像素值，最终输出恢复后的完整图像。

**核心思路**  
1. **噪声识别**：首先区分正常像素与异常像素（如值超出合理范围、与周围像素差异过大）。  
2. **插值恢复**：对异常像素，采用周围有效像素的加权平均进行填充。常见方法包括均值滤波或中值滤波，以平滑局部区域。  
3. **边界处理**：对图像边缘的异常像素，仅使用已存在的邻域像素计算，避免越界。  
4. **迭代优化**：若噪声密集，可多次迭代逐步恢复，直至图像稳定。

**关键步骤**  
- 遍历图像每个像素，记录异常位置及值。  
- 以滑动窗口（如3×3）遍历，计算窗口内有效像素的均值或中值，替换异常值。  
- 输出修复后的完整灰度图数据。

**复杂度控制**  
算法时间复杂度为O(n)，适合大规模图像处理。实际应用中可调整窗口大小或滤波策略，平衡恢复精度与计算效率。

## 205. 2024D-田忌赛马

🔴 **困难** | 标签：华为OD, 回溯

**田忌赛马问题解析（2024D）**

本题核心是“最大化胜场”，经典解法有贪心与回溯两种。贪心法（先比较双方最慢的马，若田忌的更慢则去消耗对方最强的马）在数据严格有序时有效，但对一般情况可能非最优，不推荐作为面试首选解法。

**推荐解法：回溯法（搜索所有可能）**
1. **预处理**：将齐威王和田忌的马匹速度分别排序，通常按升序排列。
2. **回溯函数设计**：
   - **参数**：当前可比较的齐马索引（`king_idx`）、田马索引（`tian_idx`）、已用田马的标记数组、当前已获分数、最大分数引用。
   - **终止条件**：当索引越界时，更新最大分数。
   - **核心逻辑**：遍历所有**未使用**的田马，与当前齐马比赛。
     - 若田马速度 > 齐马速度，得分+1，递归后回溯。
     - 无论胜负，都尝试用当前田马与下一匹齐马比赛（模拟“弃子”策略），递归后回溯。
3. **关键优化**：排序后，回溯时可按顺序尝试，利用剪枝避免无效搜索。

**时间复杂度**：O(N!)，因马数较少（通常N≤6），实际可行。
**空间复杂度**：O(N)，用于存储马匹状态。

**面试回答要点**：
- 明确指出贪心法的局限性。
- 清晰描述回溯的递归结构与决策树。
- 强调通过排序和状态管理来系统化搜索。
- 提及复杂度并说明在小数据规模下适用。

**答案示例**：
“田忌赛马问题本质是在给定两组数据中，通过匹配策略使胜利场次最大化。我的思路是采用回溯法：首先将双方马匹按速度排序，然后递归地尝试所有匹配组合。在每一步，我让田忌的一匹马与齐威王当前最慢的马比赛，若能赢则直接得分并进入下一轮；若不能赢，则选择用田忌最慢的马去消耗齐威王当前最快的马，再进入下一轮。这个过程通过递归和状态回溯来探索所有可能性，最终记录并返回最高得分。这种方法虽然时间复杂度为阶乘级，但因为马匹数量有限，在面试场景下是可行且易于理解的。”

## 206. 2024D-石头剪刀布游戏

🔴 **困难** | 标签：华为OD, 哈希表

**题目分析：**

石头剪刀布游戏的核心是建立三种手势之间的胜负关系。使用哈希表可以高效存储和查询这种映射关系。

**解题思路：**

1. **建立胜负映射：** 使用哈希表存储每个手势的"克星"，例如 `{'石头': '剪刀', '剪刀': '布', '布': '石头'}`，这样只需 O(1) 时间即可判断胜负。

2. **游戏逻辑：** 比较两人出的手势：
   - 相同：平局
   - 当前手势的克星恰好等于对方手势：当前玩家获胜
   - 否则：对方获胜

3. **代码实现：**

```python
def judge(p1, p2):
    beat_map = {'石头': '剪刀', '剪刀': '布', '布': '石头'}
    if p1 == p2:
        return '平局'
    elif beat_map[p1] == p2:
        return '玩家1获胜'
    else:
        return '玩家2获胜'
```

**复杂度分析：**

- 时间复杂度：O(1)，哈希表查找为常数时间
- 空间复杂度：O(1)，仅存储固定数量的映射关系

**扩展思考：** 若扩展到"石头剪刀布蜥蜴斯波克"等变体，只需修改哈希表的映射关系，核心逻辑不变，体现了哈希表解耦数据与逻辑的优势。

## 207. 2024D-移动元素获得最大数组和

🔴 **困难** | 标签：华为OD, 模拟

本题要求通过移动数组元素来最大化数组总和。核心思路是通过排序和贪心策略，将最大元素集中到数组前部。

**解题思路：**
1.  **问题转化**：我们可以在执行移动操作前，先对整个数组进行升序排序。这样，无论后续如何移动，我们总是可以确定哪些是较小的元素，哪些是较大的元素。
2.  **关键观察**：要使总和最大，应尽可能多地保留较大的元素。移动操作实质上允许我们选择性地“舍弃”或“保留”元素。
3.  **贪心策略**：排序后，数组前部是较小元素。我们可以决定一个“舍弃”位置 `k`（0 <= k < n），即舍弃从 `0` 到 `k-1` 共 `k` 个最小元素，保留从 `k` 到 `n-1` 共 `n-k` 个最大元素。
4.  **计算与比较**：遍历所有可能的 `k`（包括舍弃0个元素的情况），计算保留部分的元素和（即后缀和），取其中的最大值。这个最大值即为题目所求。

**算法步骤：**
1.  输入数组并进行升序排序。
2.  计算数组的后缀和数组 `suffix_sum`，其中 `suffix_sum[i]` 表示从索引 `i` 到末尾所有元素之和。
3.  初始化 `max_sum = suffix_sum[0]`（即保留全部元素）。
4.  遍历 `k` 从 `1` 到 `n`（表示舍弃前 `k` 个元素），比较并更新 `max_sum = max(max_sum, suffix_sum[k])`。注意 `k=n` 时舍弃所有元素，和为0。
5.  返回最终的 `max_sum`。

该方法通过排序和后缀和计算，高效地枚举了所有最优解情况，时间复杂度主要为排序的 O(n log n)。

## 209. 2024D-贪心歌手

🔴 **困难** | 标签：华为OD, 贪心

这道题可以使用贪心算法求解。核心思路是：在每一步，我们都选择当前可以唱的、且能让我们跳到最远位置的那首歌。

具体实现步骤如下：
1. 初始化一个变量记录当前能到达的最远位置`farthest`，以及当前遍历到的位置`current_end`。
2. 从第一首歌开始遍历数组，对于每一首歌`i`：
   * 更新`farthest = max(farthest, i + nums[i])`，其中`nums[i]`表示唱这首歌能获得的额外跳跃能力。
   * 如果`i`等于`current_end`，说明我们已到达当前所能覆盖的边界，需要进行一次跳跃（即选择一首歌来唱）。这时我们将`current_end`更新为`farthest`，并增加跳跃计数。
   * 如果在遍历过程中，`farthest`已经大于或等于数组末尾，则提前结束。
3. 最终，最少的演唱次数就是跳跃次数。

**复杂度**：算法只需遍历一次数组，时间复杂度为O(n)，空间复杂度为O(1)。

**关键点**：贪心的策略在于，我们总是希望在必须做出选择（跳跃）时，选择那个能为下一步带来最远可能性的歌曲，从而用最少的步数（演唱次数）到达终点。
