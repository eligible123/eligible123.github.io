Nginx配置：退出系统失败问题解决

发布于 2025-05-09 | 分类: 运维 | 标签: Nginx HTTPS 重定向

退出系统失败

当部署完系统后，点击退出时报错，可参照下面的说明：

1. https下admin端重定向丢失端口或者转入http协议

如果nginx前面还有多层代理，对于https加非80端口访问系统的，可以固化 X-Forwarded-Proto，X-Forwarded-Port

proxy_set_header   Host               $host;

proxy_set_header   X-Real-IP          $remote_addr;


proxy_set_header   X-Forwarded-For    $proxy_add_x_forwarded_for;


proxy_set_header   X-Forwarded-Proto  https;


proxy_set_header   X-Forwarded-Port   7080;

如果还有问题，把 proxy_set_header Host $host; 屏蔽：

proxy_set_header   X-Real-IP          $remote_addr;

proxy_set_header   X-Forwarded-For    $proxy_add_x_forwarded_for;


proxy_set_header   X-Forwarded-Proto  https;


proxy_set_header   X-Forwarded-Port   7080;

2. nginx外面还有LB时，端口不一致，根目录重定向端口不一致（丢失）问题

1. 对外对内使用相同的端口号，如都修改成8181或者81

2. nginx配置添加 absolute_redirect off;，可以取消绝对路径的重定向

server {

    absolute_redirect off;    #取消绝对路径的重定向


    ...


}
