云枢表单、流程数据迁移

项目背景

项目上需要把一个第三方系统的业务数据、已完成的流程数据迁移到云枢，包含表单数据、已办数据。

实现思路

弄清楚第三方系统的数据存储结构以及对应云枢数据存储结构的对应关系，将第三方系统数据表导入至云枢数据库，编写SQL脚本插入数据至云枢对应的数据表。

数据核对

双方数据库表核对需要注意的点：

  • 第三方系统业务数据表字段与云枢字段对应关系。
  • 注意第三方系统的用户、部门数据与云枢用户控件、部门控件的数据结构差异。
  • 第三方系统流程数据表字段与云枢字段对应关系。
  • 第三方系统流程数据与业务数据对应关系。

    • 云枢流程数据表：

    | 表名称 | 描述 | | | biz_workflow_instance | 存储流程实例数据，每一条业务数据对应一条流程实例数据 | | | biz_workitem_finished | 已办表，用户已经完成的待办会存储到这张表中，一个用户的一个已办任务对应这张表的一条数据，多条已办数据关联一条流程实例数据 | | | biz_workitem | 待办表，用户待办会存储到这张表中，一个用户的一个待办任务对应这张表的一条数据，多条待办数据关联一条流程实例数据 | | | biz_workflow_token | 审批记录表，用户对流程的审批记录会存储到这张表中，多条审批记录数据关联一条流程实例数据 | |

    流程数据迁移

    建议流程迁移顺序：

  • 迁移流程实例表（biz_workflow_instance）数据。
  • 关联流程实例数据和业务表数据。
  • 迁移已办表（biz_workitem_finished）数据。
  • 迁移审批记录表（biz_workflow_token）数据。
