云枢定时任务统一维护

一、背景需求

在云枢实际应用中，经常会设置很多定时任务，产品自带的业务定时，以及一些和表单业务无关的综合定时任务，这些任务设置了不同的时间点执行，然而有时候我们开发会本地启动服务调试，可能就遇到定时任务本地执行，一般本地配置了测试环境对接地址，然而连接生产库，导致数据错乱，在此情况下，计划定时任务统一管理，只在liunx环境运行，这样就避免数据错乱，再考虑到业务修改定时任务时间，每次都需要重启服务器，非常麻烦

二、思路

1、用云枢表单创建一个配置表，用来配置每个定时任务的名称，以及定时corn，我们只需要修改corn就可以让下次定时生效，无需重启

三、实现步骤

1、创建一个表单，配置定时名称，corn，以及备注

2、后端代码，这里直接实现SchedulingConfigurer 类，重写configureTasks方法

3、代码展示逻辑

@Component @Slf4j @EnableScheduling public class TradeCaravanTpTask implements SchedulingConfigurer {

@Autowired protected AttendanceController attendanceController;

@Autowired private DubboConfigService dubboConfigService;

@Override public void configureTasks(ScheduledTaskRegistrar scheduledTaskRegistrar) { String cornName = "商旅特批推送PM定时"; scheduledTaskRegistrar.addTriggerTask(() -> process(), triggerContext -> { BizObjectQueryModel bizObjectQueryModel = new BizObjectQueryModel(); String schemaCode = "scheduled_task"; bizObjectQueryModel.setSchemaCode(schemaCode); bizObjectQueryModel.setQueryCode(schemaCode); FilterExpression filterExpression0=new FilterExpression.Item("task_name", FilterExpression.Op.Eq,cornName); bizObjectQueryModel.setFilterExpr(filterExpression0); Page bizObjectModelPage = dubboConfigService.getBizObjectFacade().queryBizObjects(bizObjectQueryModel); //获取定时corn String cron = (String) bizObjectModelPage.getContent().get(0).getData().get("cron_expression"); return new CronTrigger(cron).nextExecutionTime(triggerContext); }); }

四、总结

此功能可以统一定时任务管理，修改定时时间无需重启服务器，也便于维护，降低维护成本，也能解决开发数据错乱问题
