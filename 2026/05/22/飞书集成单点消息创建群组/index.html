    • 前提条件

    了解飞书用户身份体系

    飞书在不同的层次对用户身份做了隔离，以保证用户的数据安全。整体来说用户身份 ID 有以下几种类型：

    - 物理用户身份（lark_id），即全局 ID - 用户在租户内的身份（user_id） - 用户在应用内的身份（open_id ） - 用户在同一应用服务商所开发的多个应用下的统一身份（ union_id）

    ​

    考虑到一个租户下可能存在多个应用，并且应用可能涉及到组织人员管理场景（创建成员/组/角色），所以使用 user_id。

    飞书相关文档https://open.feishu.cn/document/home/user-identity-introduction/introduction

    组织同步

    - 提前做好业务系统和飞书的组织架构同步，确认好业务系统使用哪个字段存储飞书的 user_id 。

  • 添加机器人能力，用于给用户发消息。

    配置内容样例

    配置内容样例只是提供一种从 模板配置 转换成 具体展示给用户的文本 的方案，仅供参考。

    模板的内容构成：

  • isDataItem（枚举值）：

    • 0：普通字符串，code 配置什么内容，就展示给用户什么内容。

    1：需要替换的字符串，code 配置的内容需要替换为实际需要展示给用户的内容，通常是一个字段编码。

    2：会被转换为换行符。

  • code：配置的内容。

    飞书工作台单点

    在 2.3. 创建飞书应用 配置首页地址时，使用 获取登录预授权码 接口地址，该接口的 redirect_uri 参数为 Encode 处理后的项目首页地址，从飞书工作台点击应用时，会自动跳转到重定向 URL，并且携带飞书的预授权码 code，根据预授权码再调用 获取 user_access_token 接口获取飞书 user_id。

    飞书消息卡片单点

    飞书消息卡片点击后，打开浏览器并跳转到目标页面，实现方式很多，例如消息卡片跳转地址填写 获取登录预授权码 接口地址，或者先判断项目中用户是否已经登录（浏览器中是否有 Token 信息），若未登录，再跳转至 获取登录预授权码 接口地址等等... 总体思路还是根据飞书的预授权码来获取用户信息完成单点。
