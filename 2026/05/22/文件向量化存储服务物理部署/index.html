文件向量化存储服务物理部署

文件向量化存储服务部署

- 需求

8.5版本需要文件向量化存储数据对知识库进行数据存储，客户需要物理部署文件向量化存储服务。

  • 部署文档
  • 关闭防火墙与SELinux

    • # 关闭防火墙

    systemctl stop firewalld && systemctl disable firewalld

    # 关闭SELinux

    setenforce 0 && sed -i 's/SELINUX=enforcing/SELINUX=disabled/' /etc/selinux/config

  • 时间同步

    • # 时间同步

    yum install -y chrony

    systemctl start chronyd.service && systemctl enable chronyd.service

  • 安装PostgreSQL16

    • 在根目录下创建/data/postgresql目录

    # 创建安装目录

    mkdir -p /data/postgresql

    #给data目前添加权限

    chmod -R 755 /data

    将安装包中的postgresql-16.10.tar.gz上传至服务器的postgresql目录

  • 解压源码

    • # 解压源码包

    tar -zxvf postgresql-16.10.tar.gz

  • 安装编译运行需要的组件

    • # 必要的组件安装

    yum install gcc gcc-c++ -y

    yum install readline-devel -y

  • 进入解压目录，检测配置

    • # 进入解压目录

    cd postgresql-16.10/

    # 配置检查

    ./configure

  • 编译安装

    • # 编译安装

    make -j4 & make install

  • 配置环境变量

    • # 编辑环境变量

    vim /etc/profile

    在文件最后增加

    # 环境变量新增内容

    export PATH=$PATH:/usr/local/pgsql/bin

    export PGDATA=/data/postgresql/postgresql-16.10/data

    export LD_LIBRARY_PATH=/usr/local/pgsql/lib:$LD_LIBRARY_PATH

    # 环境变量新增内容结束

    保存后执行下面命令使环境变量生效

    # 重新加载环境变量

    source /etc/profile

  • 创建数据目录，和用户并赋予权限

    • # 创建目录

    mkdir -p /data/postgresql/data

    mkdir -p /data/postgresql/logs

    mkdir -p /data/postgresql/archive

    mkdir -p /data/backup

  • 新增用户并设置密码

    • # 新增Linux用户postgres

    useradd postgres

    # 设置postgres用户密码

    passwd postgres

    在弹窗内输入postgres的密码并保存。

  • 修改目录权限

    • # 修改目录权限

    chown -R postgres:postgres /usr/local/pgsql/

    chown -R postgres:postgres /data/postgresql/data

    chown -R postgres:postgres /data/postgresql/logs

    chown -R postgres:postgres /data/postgresql/archive

    chown -R postgres:postgres /data/backup/

  • 修改用户环境变量

    • # 切换Linux服务器postgres用户

    su postgres

    # 编辑用户环境变量

    vi ~/.bash_profile

    # 环境变量新增内容

    export PATH=$PATH:/usr/local/pgsql/bin

    export PGDATA=/data/postgresql/data

    # 环境变量新增内容结束

    # 重新加载用户环境变量

    source ~/.bash_profile

  • 初始化数据库

    • 请确认使用的是postgres用户，如果不是，请切换postgres用户

    # 切换Linux服务器postgres用户

    su postgres

    # 初始化数据库

    /usr/local/pgsql/bin/initdb -D /data/postgresql/data/

  • 修改配置

    • # 编辑数据库配置文件pg_hba.conf

    vim /data/postgresql/data/pg_hba.conf

    # 需要修改的

    host all all 0.0.0.1/0 password

    # 需要新增的

    host replication repluser 192.168.154.187/32 md5

    # 修改数据库配置文件postgresql.conf

    vim /data/postgresql/data/postgresql.conf

  • 新增系统服务

    • 注意：必须以root账号执行

    # 新增系统命令

    vim /etc/systemd/system/postgresql-16.service

    具体命令内容在【非容器部署\postgresql\postgresql配置文件\postgresql-16.service】中

  • 修改权限并重新加载系统服务

    • # 修改执行权限

    chmod 644 /etc/systemd/system/postgresql-16.service

    #重新加载系统服务

    sudo systemctl daemon-reload

  • 启动服务

    • 注意：必须以root账号执行

    # 启动postgreSQL服务

    sudo systemctl start postgresql-16

    # 查看服务状态

    sudo systemctl status postgresql-16

    出现active (running)即表示启动成功

    # 设置开机启动

    sudo systemctl enable postgresql-16

    # 查看设置状态

    systemctl is-enabled postgresql-16
