生成历史表单数据

业务场景

在企业信息化系统中，表单数据在特定状态（如“已生效”）后需要归档为历史数据，以便于后续查询、审计或统计分析。历史数据的生成需要保留主表和子表的字段数据（排除系统默认字段），并标记为历史数据后保存到系统中。

核心需求：

- 筛选符合条件的表单数据（状态为“已生效”且未归档）。 - 将符合条件的表单数据转换为历史数据，保留非默认字段。 - 处理主表和子表数据，确保数据完整性。 - 保存历史数据并标记状态为“已完成”。

实现思路

- 筛选符合条件的表单数据：通过查询条件获取状态为“已生效”且未归档（isHistory 为“否”）的表单数据。 - 获取表单字段信息：获取主表和子表的非默认字段，用于数据提取和处理。 - 处理单条数据：

- 遍历主表字段，复制非默认字段数据。 - 处理子表数据，复制子表非默认字段数据。 - 标记数据为历史数据（isHistory = “是”）。

- 保存历史数据：使用原创建人 ID 保存历史数据到系统，并设置状态为“已完成”。 - 异常和日志管理：确保处理过程中记录详细日志，捕获异常并抛出业务异常。

实现步骤

1. 生成历史表单数据

说明：

- 使用 FilterExpression 构造查询条件，筛选状态为“已生效”且未归档的数据。 - 如果没有符合条件的数据，抛出异常并记录日志。 - 使用 childPropertyCache 缓存子表字段，优化性能。

2. 获取非默认字段列表

说明：

- 通过 BizPropertyFacade 获取表单的所有字段。 - 使用 Stream API 过滤掉默认字段（如系统字段），仅保留业务字段。

3. 处理单条表单数据

说明：

- 遍历非默认字段，分别处理主表字段（直接复制）和子表字段（调用 processSheetData）。 - 使用 childPropertyCache 缓存子表字段，减少重复查询。 - 标记历史数据（isHistory = “是”）并设置状态为 COMPLETED。 - 检查创建人信息，确保保存时使用正确的用户 ID。

4. 处理子表数据

说明：

- 遍历子表数据，复制非默认字段到新的历史数据对象。 - 确保只处理业务字段，排除系统默认字段。

实现效果

  • 功能性：

    • - 成功筛选出状态为“已生效”且未归档的表单数据。 - 正确处理主表和子表字段，仅保留非默认字段，生成历史数据。 - 保存历史数据并标记为“已完成”，确保数据可追溯。

  • 性能：

    • - 使用 childPropertyCache 缓存子表字段，减少重复查询，提升性能。 - 对于小型表单（数据量少、子表行数少），处理速度快；但对于大量数据或复杂子表，可能需要进一步优化。

  • 用户体验：

    • - 日志详细记录了处理过程（包括数据量、字段数、成功/失败信息），便于调试和监控。 - 异常处理机制确保数据处理失败时抛出明确错误信息，方便问题定位。

  • 健壮性：

    • - 检查空数据、缺失创建人等异常情况，确保流程健壮。 - 使用 try-catch 捕获处理过程中的异常，避免程序中断。
