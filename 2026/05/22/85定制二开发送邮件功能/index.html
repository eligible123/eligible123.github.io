8.5定制二开发送邮件功能

一、接口核心思路梳理

需要整理的这个邮件发送服务（EmailServiceImpl）实现的企业级邮件发送接口，核心目标是从业务系统中读取邮件模板和业务数据，替换模板占位符后发送带动态内容和附件的邮件，整体思路可拆解为以下 6 个核心步骤：

1. 初始化邮件发送客户端

通过@PostConstruct注解的init()方法完成 JavaMailSenderImpl 的配置，包括：

- 阿里云企业邮箱 SMTP 服务器参数（地址、端口、账号密码） - SSL/TLS 安全连接配置（解决证书校验、协议兼容问题） - 编码配置（解决邮件内容 / 附件名乱码）

2. 核心发送流程

3. 关键数据来源

- 邮件模板数据：通过getTemplateData()从 CloudPivot 引擎查询指定编码的模板（含邮件主题、正文模板） - 业务对象数据：通过getBizObjectData()从 CloudPivot 引擎查询具体业务单据数据（用于替换模板占位符） - 附件数据：从业务数据中解析附件信息，通过文件 ID 从 CloudPivot 文件存储下载附件并添加到邮件

4. 动态内容处理

通过replacePlaceholders()方法解析模板中的${变量名}占位符，支持：

- 普通字符串 / 数值替换（数值自动保留 2 位小数） - 特殊变量sendTime自动替换为当前日期（yyyy-MM-dd）

5. 异常处理

- 分类捕获 MessagingException（邮件发送异常）、IllegalArgumentException（参数异常）等 - 完整的日志记录（发送开始 / 成功 / 失败），便于问题排查

6. 附件处理

- 支持从业务数据中读取附件列表（JSON 格式） - 下载附件并转换为 ByteArrayDataSource，解决附件名乱码问题 - 异常时仅记录日志，不影响主邮件发送流程

5. 扩展建议

- 配置外置化：将 SMTP 服务器、账号密码等配置移到 application.yml，避免硬编码 - 批量发送：支持收件人列表批量发送，优化性能 - 重试机制：添加邮件发送失败重试逻辑（如使用 Spring Retry） - 异步发送：通过 @Async 注解实现异步发送，避免阻塞主线程 - 模板缓存：缓存邮件模板数据，减少数据库查询

三、总结

核心要点回顾

- 核心流程：模板读取 → 数据替换 → 邮件构建 → 发送，基于 CloudPivot 引擎实现业务数据与邮件模板解耦； - 复用代码：邮件客户端初始化、占位符替换、附件处理是通用模块，可适配不同邮箱 / 文件存储场景； - 关键优化：解决了 SSL 协议兼容、编码乱码、附件处理等企业级邮件发送的常见问题。

关键注意事项

- 使用时需替换 SMTP 服务器配置和 CloudPivot 引擎相关调用； - 附件处理需保证文件 ID 有效性，异常时仅记录日志不影响主流程； - 数值型占位符会自动保留 2 位小数，特殊变量（如 sendTime）可按需扩展。
