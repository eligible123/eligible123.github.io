三.解决方案

针对linux服务器缺少中文相关字体问题，⾸先在windows的 '控制⾯板'->'字体' C:\Windows\Fonts中，将⾃⼰需要的字体复制出来。保存到一个文件夹中。把文件夹通过ftp传输到Linux中的/usr/share/fonts/下⾯。例如文件夹名为fonts。完成上述操作之后，现提出两种解决方案：

1.第一种方案是通过代码层面解决，首先引入开包工具包依赖，加入以下这段代码，该段代码是指定linux服务器下中文字体的文件目录，进行导出。

2.第二种方案是在linux服务器上直接缓存windows的中文字体

① 打开linux终端，进⼊root状态。在/usr/share/fonts/目录下输⼊ chmod 775 /usr/share/fonts/fonts 命令。

② 然后输入下面命令： cd /usr/share/fonts/fonts sudo mkfontscale sudo mkfontdir sudo fc-cache -fv

③ 安装字体索引指令：

使mkfontscale和mkfontdir命令正常运行，输入 yum install mkfontscale 命令。

使fc-cache命令正常运行，如果提示 fc-cache: command not found，输入 yum install fontconfig 命令。

④ 查看Linux服务器已安装中文字体，输入 fc-list :lang=zh 命令。
