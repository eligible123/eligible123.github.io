window开启ssh安全访问

在 Windows 上安装和配置 OpenSSH 服务器可以让你将 Windows 机器作为 SSH 远程访问的目标。以下是详细的安装和配置步骤：

1. 检查 OpenSSH 是否已安装

Windows 10（1809+）和 Windows 11 内置了 OpenSSH 客户端和服务器组件，但默认可能未安装。

查看已安装的 OpenSSH 组件（以管理员身份运行 PowerShell）：

  • Get-WindowsCapability -Online | Where-Object Name -like 'OpenSSH*'
  • 如果 State 为 NotPresent，表示未安装。

    在另一台机器（如 Linux 或另一台 Windows）上使用 SSH 客户端连接：

  • ssh username@你的WindowsIP

    其他问题处理

    当 OpenSSH.Server 在 Windows 系统上显示为 "InstallPending" 状态时，通常表示安装或卸载操作尚未完成，可能由于系统 pending 操作（如挂起的重启或依赖进程未完成）。以下是解决方法：

    1. 重启系统

    这是最常见且有效的解决方案，因为某些安装步骤需要重启才能完成：

  • 保存工作，关闭所有程序。

    • 以管理员身份运行命令提示符或 PowerShell，执行：

  • shutdown /r /t 0

    • 重启后检查状态：

  • Get-WindowsCapability -Online | Where-Object Name -like 'OpenSSH.Server*'

    • 2. 手动完成安装

    如果重启无效，尝试手动完成安装：

    通过 PowerShell 完成安装：

    通过 DISM 工具修复：

    DISM /Online /Add-Capability /CapabilityName:OpenSSH.Server~~~~0.0.1.0

    3. 检查挂起的操作

    某些系统操作（如更新或软件安装）可能阻塞了 OpenSSH 的配置：

    以管理员身份运行 PowerShell，检查挂起的重启：

  • Test-Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\RebootPending"

    • 如果返回 True，则必须重启。

    清理挂起的操作：

  • Remove-Item -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\RebootPending" -Force -ErrorAction SilentlyContinue

    • （谨慎操作，建议优先重启）

  • 打开 事件查看器 > Windows 日志 > 应用程序，筛选来源 DISM 或 Windows-Servicing 的错误。

    • 6. 重置 Windows Update 组件

    如果问题与更新相关：

    总结步骤

    - 重启电脑 → 检查状态。 - 若未解决，手动重装 OpenSSH.Server。 - 检查挂起的操作/日志，排除冲突。 - 仍失败可尝试 DISM 或系统修复。

    如果问题持续，可能需要更深入的系统检查（如 SFC 扫描或系统还原）。
