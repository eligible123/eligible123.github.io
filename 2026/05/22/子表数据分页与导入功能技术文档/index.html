子表数据分页与导入功能技术文档

1. 概述

本文档详细说明子表数据的分页展示、分页事件处理、数据导入覆盖及提交流程的技术实现方案。旨在确保子表数据展示与操作的正确性和一致性，提升用户体验和数据管理效率。

2. 功能需求

  • 存储子表原始数据，支持分页展示。
  • 根据后台配置的默认分页条数，切割数据并展示。
  • 保持分页数据结构与原始数据结构一致。
  • 实现分页点击事件，动态切割数据并刷新子表。
  • 清空子表数据后赋值，确保数据长度与分页条数一致。
  • 处理分页切换时的选中状态清除。
  • 支持导入数据覆盖现有数据，字段逐一替换，避免直接覆盖。
  • 修改子表字段时触发产品底层逻辑块进行相应处理。
  • 提交时将全量数据传给后端，由后端完成数据修改。

  • 在组件或状态管理中定义变量 sheetOriginalData，用于存储子表所有数据。
  • 该数据作为分页切割的基础。

    updateOriginalArray(originalArr, modifyArr, idField) {

    const modifyMap = new Map();

    modifyArr.forEach(item => {

    modifyMap.set(item[idField], item);

    });

    originalArr.forEach(originalItem => {

    3.8 提交数据

    - 目标**：提交时将完整数据传给后端，由后端完成数据更新。

  • *实现*：
  • 提交时，直接传递 originalData。
  • 后端根据业务逻辑处理数据修改和持久化。

    • 4. 注意事项

  • 切割数据前务必清空子表数据，避免数据残留。
  • 替换导入数据时，避免直接替换对象引用，防止数据绑定异常。
  • 分页切换时清除选中状态，保证用户操作一致性。
  • 后端接口需支持接收全量数据，完成数据更新。

    • 5. 总结

    通过以上设计，子表数据分页与导入覆盖功能实现了数据的完整管理和用户友好交互，保证了数据一致性和操作的可控性。
