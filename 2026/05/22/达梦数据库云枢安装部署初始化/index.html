达梦数据库云枢安装部署初始化

安装

本次安装服务器是【麒麟】，虚拟机为Linux

达梦数据库是收费的，别自己安装了。找厂商

达梦安装会有很多设置都会帮你调整好。（列如：大小写忽略，防火墙，备份策略...）

建完库之后。会建立对应的用户与对应表空间，以及给定权限

> #建表空间

create tablespace cloudpivot datafile 'cloudpivot.dbf' size 4096 AUTOEXTEND on;

create TABLESPACE H3YUN_REPORT DATAFILE 'H3YUN_REPORT.DBF' SIZE 4096 AUTOEXTEND on;

#建用户并控制对应的表空间

CREATE USER cloudpivot IDENTIFIED BY "test123456!@#$%" DEFAULT TABLESPACE cloudpivot DEFAULT INDEX TABLESPACE cloudpivot;

CREATE USER h3yun_report IDENTIFIED BY "test123456!@#$%" DEFAULT TABLESPACE H3YUN_REPORT DEFAULT INDEX TABLESPACE H3YUN_REPORT;

#赋予对应的权限

grant resource,vti,soi to cloudpivot;

grant resource,vti,soi to h3yun_report; ​

启动与停止

> #进入达梦安装的bin目录下

cd /dmdbms/bin

#启动

./DmServiceDMSERVER start

#停止

./DmServiceDMSERVER stop 如果有的地方不能通过这种方式启动服务

可以在找到安装路径执行，通过前台启动的，退出之后就关闭了

> #通过前台启动的方式，路径明确后执行

/dm8/bin/dmserver /dm/data/DMDB/dm.ini 备份与还原

备份可以通过可视化工具【DM管理工具】在线备份。安装好达梦数据库之后，在安装目录的doc下会有很多操作手册。写的很繁琐，但是比在网上找对应的操作友好一点，信创环境，网上找到的资源还是少

右键新增备份即可。

还原装在虚拟机Linux上的只能通过命令执行。达梦数据库提供的【DM控制台工具】不能连接远程服务器

而且还原必须要在脱机的情况下进行。本次记录的是表空间还原。

> #切换用户到dmdba

su - dmdba

#进入达梦安装的bin目录下

cd /dmdbms/bin

#停止服务

./DmServiceDMSERVER stop

#进入达梦数据库命令执行控制台 进入RMAN

./dmrman

#还原备份之前，先备份一次，这是在备份库

backup database '/dmdbms/data/DAMENG/dm.ini' backupset 'full_bak'

#退出RMAN

exit

#进入备份仓库 查看备份文件

cd /data/DAMENG/bak

#进入对应的备份

cd TS_cloudpiovt_FULL_2024_03_31-16_57_56/

#获取当前路径

pwd

/dmdbms/data/DAMENG/bak/TS_cloudpivot_FULL_2024_03_31_16_57_56

#还原表空间。启动 DMRMAN，输入以下命令

restore database '/dmdbms/data/DAMENG/dm.ini' tablespace cloudpivot from backupset '/dmdbms/data/DAMENG/bak/TS_cloudpivot_FULL_2024_03_31_16_57_56'

#恢复表空间。启动 DMRMAN，输入以下命令

recover database '/dmdbms/data/DAMENG/dm.ini' tablespace cloudpivot with archivedir '/dmdbms/data/DAMENG/arch' until time '2024-03-31 16:57:56'

#达梦官方提供的语句是

recover database '/dmdbms/data/DAMENG/dm.ini' tablespace cloudpivot

达梦DBA人员执行的是未注释的语句，最后没有什么效果。

# 说明1：DBA：备份后执行的DDL语句太多

说明2：time 对应的时间节点还能向前推

猜测：达梦数据库备份还原确实有问题

#退出DMRMAN

exit

#启动服务

./DmServiceDMSERVER start 进入数据库查看是否还原

迁移数据

迁移数据使用达梦提供的【DM数据迁移工具】

可视化操作。注意选着的配置，可提供的参考文档

就是这个地方要注意不要选着【创建模式】...达梦DBA人员没有说明为什么不选这个

另外注意的是，如果目标表空间已存在，工具并不能进行覆盖迁移，迁移之后反而会破坏表空间结构，导致想删除都删除不了

数据库关键字忽略

DM数据库在查询的时候，会出现关键字问题。

例如查询人员表时admin是关键字段，这句查询会报错。是因为【admin】在达梦数据库中是关键字。需要修改配置进行忽略

> select admin from h_org_user limit 100 修改方式1：

找到安装目录下的dm.ini文件

打开文件，找到EXCLUDE_RESERVED_WORDS，添加要忽略的关键字 XML,EXCHANGE,DOMAIN,link

修改方式2：

> SP_SET_PARA_STRING_VALUE(2, 'EXCLUDE_RESERVED_WORDS','XML,EXCHANGE,DOMAIN,link'); 这两种修改方式选一直即可。修改完成之后要重启数据库

数据库连接器

目前有使用到两个：DM管理工具（官方）、DBeaver

在这两个连接工具中都遇到了问题，有的SQL无法执行。举例：DDL语句中给字段类型int（INT），构建函数参数类型int（INT）

有的在一个上执行不了，在另外一个上能执行

有的都执行不了，就要手动去编辑修改...
