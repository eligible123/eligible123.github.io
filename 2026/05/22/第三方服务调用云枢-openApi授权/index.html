第三方服务调用云枢 openApi

场景

第三方服务需要调用云枢 openApi，使用云枢配置实现。配置完成后，客户端使用设置好的 clientId 和生成好的 clientSecret 请求云枢接口获取 token。

方法与步骤

  • 使用 admin 账号登录云枢，后台管理——》系统管理——》API接口配置，新增一条数据。

    • 参数说明

    authorities：表示第三方需要请求的接口类型

    clientId：标识第三方客户端

    ClientSecret：生成好的密钥，需要保存在第三方客户端

    authorizedGrantTypes：授权类型

    registeredRedirectUris：重定向url

       accessTokenValiditySeconds： 访问令牌有效时间

    refreshTokenValiditySeconds：刷新令牌有效时间

  • 配置完成后，请求token接口获取token（接口不用开发，云枢自带）

    • 请求说明

    | 字段 | 值 | 描述 | | | 请求类型 | POST | | | | 请求地址 | /api/oauth/token | | | | 响应类型 | JSON | | |

    • 注意事项

    参数设置中， accessTokenValiditySeconds（访问令牌有效时间 ）和 refreshTokenValiditySeconds（刷新令牌有效时间）的设置应当遵循：刷新令牌有效时间 >= 访问令牌有效时间，由于 Oauth2 的令牌机制，访问令牌 过期后，只需要使用 刷新令牌 和 clientSecret 重新获取一个 访问令牌 就可以继续访问资源，而只有当 刷新令牌 过期后，用户才需要重新登录系统，过长的访问令牌有效时间可能导致访问令牌泄露，从而引起安全问题，所以尽量不要将访问令牌的有效时间设置的过长。
