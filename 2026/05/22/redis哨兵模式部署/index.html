redis哨兵模式部署

前言

Redis 哨兵模式（Sentinel）是基于主从复制的高可用解决方案，核心目标是解决主节点故障后的自动恢复问题，当主节点（Master）因故障下线时，哨兵会自动检测到故障（通过心跳机制），并从从节点（Slave）中选举一个新的主节点，同时更新其他从节点的复制目标，实现服务无缝切换，无需人工干预。相较于集群模式只需额外部署哨兵进程并配置监控规则，无需复杂的分片逻辑。适合中小规模场景，对高可用有要求但数据量未突破单节点承载能力的业务。
