带鉴权的 Webservice 接口调用

需求

某项目中，需要对接第三方 WebService 接口，且接口带鉴权逻辑，第三方接口基于 SOAP 协议。

实现思路

什么是 WebService 接口

Web Service 接口是一种允许不同系统之间进行通信和数据交换的技术。它们提供了一种标准化的方式来使应用程序能够通过网络相互通信，而无需考虑它们所使用的编程语言或操作系统。

什么是 SOAP 协议

SOAP (Simple Object Access Protocol) 是一种基于 XML 的消息传递协议，它定义了消息的结构、编码规则以及调用和响应模式。SOAP 具有较为严格的规范和标准，因此在企业级应用中被广泛使用。

调用 WebService 接口的要素

基于 SOAP 协议的 WebService 接口可以通过 HTTP 进行请求，因为 SOAP WebService 使用 HTTP 或者 HTTPS 作为传输协议，我们可以将 SOAP 格式的 XML 放在 RequestBody 中，发送 POST 请求即可。

​

所以在调用接口之前，我们需要明确以下几点：

  • 接口地址
  • 请求体内容 （XML）
  • SOAPAction
  • 接口鉴权方式

    • 在此次项目中：

    - 接口地址： http://eip.mmsh.sinopec.com/xxzx/_vti_bin/authentication.asmx  （认证） http://eip.mmsh.sinopec.com/xxzx/_vti_bin/Lists.asmx （执行业务）

    ​

  • 请求体内容
  • SOAPAction

    • - 接口鉴权方式 基于 cookie ，当调用认证接口成功后，会在响应头中返回用于认证的 cookie 的值，再下次调用业务接口时，在请求头中携带即可。
