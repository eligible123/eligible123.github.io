Vue 自定义拖拽指令

一、业务场景

在课程中心项目里，为了给用户提供更优质的交互体验，需要在页面上添加一个倒计时组件，用于显示完成课程学习还需的时间。并且，这个倒计时组件应该支持用户手动拖拽，方便用户根据自己的需求调整其在页面上的位置。例如，当用户在观看视频或者阅读文档时，可以将倒计时组件拖拽到不影响视线的位置，同时又能随时看到剩余的学习时间。

二、实现思路

为了实现倒计时组件的拖拽功能，我们采用Vue 自定义指令的方式。自定义指令可以将拖拽逻辑封装起来，提高代码的可复用性和可维护性。具体思路如下：

​

1. 注册自定义指令：在 index.vue 组件中注册一个名为 drag-countdown 的自定义指令。

2. 监听鼠标/ 触摸事件：在指令的 inserted 钩子函数中，为组件添加鼠标和触摸事件监听器，以便捕获用户的拖拽操作。

3. 计算偏移量和位置：在用户开始拖拽时，计算鼠标或触摸点相对于组件的偏移量。在拖拽过程中，根据偏移量和鼠标或触摸点的位置，计算组件的新位置。

4. 限制组件位置：确保组件在拖拽过程中不会超出其父容器的范围。

5. 清理事件监听器：在组件解绑时，移除所有的事件监听器，避免内存泄漏。

4. 代码解释

· 获取容器和拖拽手柄：通过 binding.arg 获取自定义的容器元素，如果没有提供，则使用父元素作为容器。通过 binding.modifiers.handle 判断是否指定了拖拽手柄。

· 获取容器边界：通过 getContainerBounds 函数获取容器的边界信息，用于限制元素的拖拽范围。

· 限制元素位置：通过 constrainToContainer 函数将元素的位置限制在容器内。

· 更新元素位置：通过 updatePosition 函数根据鼠标或触摸事件的位置更新元素的位置。

· 事件处理：通过 startDrag、drag 和 endDrag 函数处理鼠标和触摸事件，实现元素的拖拽功能。

· 初始化样式：为元素添加 transform: translate3d(0,0,0) 样式，开启硬件加速，提高拖拽的流畅度。

· 添加事件监听：为拖拽手柄添加 mousedown 和 touchstart 事件监听器，开始拖拽操作。

· 清理函数：定义 el.__draggableCleanup 函数，用于在元素解绑时清理事件监听器。
