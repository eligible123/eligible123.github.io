金工消息中心对接文档

金工消息中心目前支持OA、钉钉、短信、邮件等消息发送。

1 接入前提

1.1 账号申请

接入前需要先申请对应用户和权限，提供接入方系统名、应用缩写。

新建成功后提供给接入方应用id、应用秘钥、发送消息用户和密码。

1.2 认证接口

消息中心采用oauth2.0标准进行认证，调用各消息接口前需要先认证通过，获取该用户access_token。

认证接口（以测试环境为例）：

POST /oauth/user/token

Request URL：http://172.16.192.41:30010/api-auth/oauth/user/token

Host:172.16.192.41:30010/api-auth

Content-Type:application/json

Query参数：

Header参数：

返回响应

OK (200)

{ "access_token": "c7beeb9f-0334-4c40-95fe-c521665baa80", "token_type": "bearer", "refresh_token": "9d0f8520-2321-4a63-9692-baa71d610cd9", "expires_in": 12551, "scope": "all"}

注意：建议对接系统可以把返回access_token缓存处理，避免多次请求。

1.3 应用租户

Header参数：

注：在低代码平台，其中一些应用需要把消息的归属到划转到其他自研的业务系统，同时把菜单挂到其他自研系统里。针对这种情况，该请求头必须要指定为对应自研系统的应用ID。
