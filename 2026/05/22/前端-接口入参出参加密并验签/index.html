前端-接口入参出参加密并验签

接口入参出参加密并验签

一、需求

客户需要提高系统安全性，加大系统被攻克难度，提出对接口入参出参进行加密，并进行验签和防重放控制。

二、思路

1、确定加密方案

2、采用拦截器对接口进行处理

三、解决方案

1、入参加密方案 采用RSA密钥对，前端随机生成AESKey，将AESKey用PubKey采用RSA加密生成SecKey，将入参data用SHA-256加密为hash值，将data，hash，timestamp作为入参data对象，再用AES加密得到SecData传给后端。后端先用PriKey对SecKey解密得到AESKey，用AESKey对SecData解密得到data，hash，timestamp，将data进行SHA-256加密和hash进行对比进行验签，避免参数修改。对timestamp+userId+方法地址用redis进行防重放验证。

2、出参加密方案 后端将返回data进行SHA-256加密得到hash，将data和hash放在同一个data对象，再对返回的data对象用SecKey进行AES加密后返回前端，前端收到之后用SecKey进行AES解密，将得到的data进行SHA-256加密和hash进行对比进行验签。如下图

3、参数说明

// data -> get 请求将url里的params变成对象放入，post请求对应body里的对象

// hash -> 将 data的值进行SHA-256加密后的字符串

// timestamp 当前时间timestamp {

‘data':{},

'hash':xxxxx;

'timestamp':''

}

避免参数名特殊给攻击者猜到用途，SecKey -> code 放于Heard中，SecData -> data 放于Heard中，同时SecData也传递在参数对应位置，及get-> /search_workitems?data=LofAjch ，post -> body中 {‘data':’ LofAjch’}

四、实现步骤

1、前端每次刷新token必须生成新的AESKey(修改所有token的文件)并且缓存;

2、请求参数前端统一拦截处理, 将get,post请求参数,使用哈希SHA-256加密, 生成数据格式是{ data: 接口原始请求参数, code: 使用哈希SHA-256加密编码 }

3. 前端请求成功后, 后端返回加密参数(加密参数和传参格式一样), 然后在前端响应体做验签处理

4. RSA公钥是统一固定, 更换需要手动去修改
