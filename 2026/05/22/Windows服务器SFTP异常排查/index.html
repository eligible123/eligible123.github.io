Windows服务器SFTP异常排查

背景

客户的服务器部署在windows上，客户重启服务器后发现SFTP在服务器使用不了，经排查发现客户的SFTP使用的是freeFTPd作为SFTP服务器端口为22，但是我打开该软件一直提示端口被占用。

排查步骤

- 使用netstat -ano | findstr ":22"查找出占用端口的进程号 - - 使用 tasklist | findstr ""查看到对应的进程名称 - 发现sshd的服务占用 - 按下 Win + R 打开运行窗口，输入 services.msc 并回车，打开 “服务” 管理界面。

  • 在服务列表中找到 SSH 服务器相关的服务：

    • - 若安装的是 OpenSSH 服务器，服务名称通常为 OpenSSH SSH Server。 - 若使用的是 freeSSHd，服务名称可能是 freeSSHd 或 sshd。

    - 右键该服务，选择 “停止” 即可临时关闭服务。 - 若想彻底禁用（开机不自动启动），右键服务 → “属性”，将 “启动类型” 改为 “禁用”，点击 “确定”。

  • 再开启freeFTPd成功
