3、基于树形控件统计方面优化

背景：

在统计分析时，需要统计分析树形接口中某一节点下所有子节点的所有数据。常规统计逻辑是找到所有子节点id,然后通过 in （ids）来处理，当数据量大，同时SQL复杂的情况，会降低查询速率。此时通过加一个冗余字段，构建全路径查询码，可通过模糊匹配 like 该节点的查询码即可一步查询所有该节点及该节点的子节点数据。

举例：

A -- A1---A11

|--A2 ---A21

| ---A22

上图要查询A2节点及其所有子节点（A2 ,A21,A22)常规是要in（A2 ,A21,A22）的id。

构建查询码

此时只需要模糊匹配 like '#a#a2%'即可查询A2节点及其所有子节点（A2 ,A21,A22)。

由于很多时候，树的层级关系很重要，为了便于查询树的节点及其所有子节点，我们可以在数据新增的时候利用JS去赋值冗余字段（类似于部门queryCode）,则可以快速通过categoryCode识别树形节点下的所有子节点，SQL通过模糊匹配categoryCode即可。

> //渲染完成后

form.on('onRendered',function(data){

var that = this;

that.parentCode.valueChange.subscribe((change)=>{

if(change.oldValue == null || change.value != change.oldValue) {

var parentCode = that.parentCode.value;

if(parentCode == "" || parentCode == "undefined" || parentCode=="null"){

that.categoryCode.value = data.id;

that.categoryParentId.value = parentCode;

} else {

that.categoryCode.value = parentCode + "_" + data.id;

}

}

});
