基于Oauth2的云枢流程触发业务服务接口配置

场景：云枢流程中触发业务服务接口，由于项目安全性要求，不允许不携带 Token 的请求；基于 Oauth2 客户端的模式实现请求接口携带 Token。

由于云枢语雀上对于 Oauth2 客户端模式下，业务服务的配置描述的不够直接，且没有相关配置实例说明，故分享一下配置步骤以及代码中需要的更改。

配置步骤

云枢管理后台配置

1. 新增客户端接入配置

- 云枢管理后台 —> 系统管理 —> API 接口配置 —> 点击新增 ​

2. 配置 clientId、authorizedGrantTypes、accessTokenValiditySeconds、refreshTokenValiditySeconds

参数说明： clientId： 系统标识，可以任意取，最好是具有某些实际意义的单词缩写。 authorizedGrantTypes： 授权类型，包含四种类型，以下是四种类型的解释，在本次实践中使用的是 客户端凭证（client credentials）

accessTokenValiditySeconds： accessToken有效时间，根据具体需求设置。 refreshTokenValiditySeconds： refreshToken有效时间，根据具体需求设置，一般情况下要长于accessToken过期时间。

3. 配置业务服务

注：客户端ID 对应上一步 新增客户端接入配置 中的clientID，客户端凭证 对应 clientSecret，获取token地址一般配置为云枢提供的获取token接口URL http://ip:port/api/oauth/token?grant_type=client_credentials&client_id=EBS&client_secret=1f64416a0f10410e8c1de53e41ebeb39。

代码修改

找到 ResourceConfig.java 类，在需要调用的接口配置后添加 hasAuthority(),至于入参是填 api 还是 openapi，取决于在 新增客户端接入配置 中勾选的 authorities 。
