云枢企业 AI 平台项目介绍

云枢企业 AI 平台前端 — 目录结构与依赖说明

版本标识：AI-Platform-v1.1.0（根 package.json）
文档用途：说明仓库目录职责、工作区内包与 npm 依赖的分类，并附带可渲染为思维导图的结构（Mermaid）。

1. 仓库与技术栈概览

维度 说明
形态 Yarn Workspaces + Lerna 的 Monorepo（根 workspaces 含 packages/、packages/builtin/、packages/plugin/*）
默认 Registry 企业 Nexus：https://nexus01.authine.cn/repository/npm-all/（.npmrc / .yarnrc）
可发布库 Registry https://nexus01.authine.cn/repository/npm-cloudpivot/（各 cloudpivot-* 的 publishConfig）
主框架 Vue 2.7、Vue Router 3、Vuex 3、TypeScript ~4.1
三端应用 admin（管理端）、portal（门户）、mobile（移动端）

2. 根目录详细结构

2.1 关键根级文件职责

路径 作用
package.json 全仓依赖锁定在根；scripts 统一调度三端 serve/build/dll；lint-staged + husky 控制提交质量
lerna.json Lerna 与 Yarn Workspaces 协同；packages 字段当前为 packages/*（builtin/plugin 由 workspaces glob 纳入）
bin/install-fix.js 子应用构建/启动前常用修复逻辑（链接、补丁类）
config/proxy.js、config/dev-config/ 本地联调后端、代理与调试入口

3. packages/ 目录结构（工作区包）

3.1 典型应用包内部（以 admin / portal / mobile 为例）

三端均为 Vue CLI 工程，常见子目录：

  • public/：静态入口、index.html、运行时 config.js 等
  • src/：路由、视图、API 适配、主题、启动 main.ts / bootstrap.ts
  • extends/：主题或扩展配置（如 theme.js、api.ts）
  • webpack.dll.config.js（若使用）：DLL 预构建加速开发

5. 根 package.json 依赖：公用 vs 私有/企业定制

  • 公用依赖：开源生态主流库，通过企业 Nexus 代理/缓存安装，版本仍遵循 semver。
  • 私有/企业定制依赖：@h3/、@h3print/、h3-* 等浩鲸/云枢体系下的 UI、报表、打印、移动端组件；通常为公司内部制品。

5.2 开发依赖 devDependencies（归类）

类别 代表依赖 作用

公用 · 语言 typescript / @babel/core TS 与转译

公用 · Monorepo lerna 多包版本与发布流程
公用 · 提交 @commitlint/* / commitizen / cz-customizable 规范化 commit message

6. 三端应用对工作区库的依赖关系（摘要）

  • admin：引入最全，包含 cloudpivot-admin-core、cloudpivot-ai、cloudpivot-web-ide、cloudpivot-designer、全量设计器与 monaco-editor 等。
  • portal：不含 admin-core / ai / web-ide，侧重门户与设计器、列表、流程、表单等。
  • mobile：子集依赖，无 cloudpivot-designer、cloudpivot-gantt、cloudpivot-kindeditor 等偏 PC 设计器类包。

7. 思维导图（Mermaid mindmap）

8. 转为外部思维导图工具的几种方式

9. 文档维护说明

  • 工作区包列表以 packages/**/package.json 为准；新增包时请同步更新本文第 3、4 节。
  • 根依赖以根 package.json 为准；升级大版本后请更新第 5 节表格中的「作用摘要」若行为有变。
  • lerna.json 中 packages 仅列 packages/* 时，builtin/plugin 仍由 Yarn workspaces 纳入安装，与 Lerna 命令范围可能不一致，排障时需注意。
